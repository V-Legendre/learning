10 — Form Validation

Zod + React Hook Form: single schema for validation and TypeScript types, field and form-level validation, async checks, and error display.

Why Zod + React Hook Form?

React Hook Form has built-in validation (required, minLength, etc.), but it's limited. Zod gives you a single schema that is both your validation rules and your TypeScript type. Define once, use everywhere.

Pydantic (Python)
from pydantic import BaseModel

class SignupForm(BaseModel):
    username: str  # min 3 chars
    email: EmailStr
    password: str  # min 8 chars

# Schema IS the type.
# Validation IS type checking.
Zod (TypeScript)
import { z } from 'zod'

const signupSchema = z.object({
  username: z.string().min(3),
  email: z.string().email(),
  password: z.string().min(8),
})

// Infer the TS type from the schema:
type SignupForm = z.infer<
  typeof signupSchema
>
The key insight: z.infer

z.infer<typeof schema> extracts the TypeScript type from a Zod schema. You never write the type manually. Change the schema, the type updates automatically. This is the same idea as Pydantic — the model definition IS the validation AND the type.

Setup

$ pnpm add react-hook-form zod @hookform/resolvers

The @hookform/resolvers package bridges Zod and React Hook Form. It provides zodResolver, which translates Zod validation errors into the format RHF expects.

import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'

const schema = z.object({
  email: z.string().email('Invalid email'),
  password: z.string().min(8, 'Min 8 characters'),
})

type FormValues = z.infer<typeof schema>

const { register, handleSubmit } = useForm<FormValues>({
  resolver: zodResolver(schema),  // <-- This connects Zod to RHF
})

Field Validation Rules

Zod provides a rich set of built-in validators. Here are the most common ones you'll use in forms:

String validations

const schema = z.object({
  // Required (non-empty)
  name: z.string().min(1, 'Name is required'),

  // Email format
  email: z.string()
    .min(1, 'Email is required')
    .email('Invalid email address'),

  // Min/max length
  username: z.string()
    .min(3, 'At least 3 characters')
    .max(20, 'Max 20 characters'),

  // Regex pattern
  slug: z.string()
    .regex(/^[a-z0-9-]+$/, 'Only lowercase letters, numbers, hyphens'),

  // URL format
  website: z.string().url('Must be a valid URL').optional(),

  // Exact value from a set (enum)
  role: z.enum(['admin', 'user', 'editor'], {
    errorMap: () => ({ message: 'Select a valid role' }),
  }),
})

Number validations

const schema = z.object({
  // Basic number
  age: z.number()
    .min(13, 'Must be at least 13')
    .max(120, 'Really?'),

  // Integer only
  quantity: z.number().int().positive(),

  // coerce: convert string input to number
  // HTML inputs always return strings, so this is essential
  price: z.coerce.number()
    .min(0, 'Price must be positive'),
})
HTML inputs return strings! Even <input type="number"> gives you a string. Use z.coerce.number() to automatically convert the string to a number during validation. Without coerce, you'll get a "Expected number, received string" error.

Optional and nullable fields

const schema = z.object({
  // Optional: field can be missing or undefined
  middleName: z.string().optional(),
  // Type: string | undefined

  // Optional with default: fill in if missing
  country: z.string().default('US'),
  // Type: string (always has a value)

  // Nullable: field can be null
  deletedAt: z.string().nullable(),
  // Type: string | null

  // Transform empty string to undefined (common for optional text inputs)
  bio: z.string().transform(v => v === '' ? undefined : v).optional(),
})

Form-Level Validation

Sometimes validation depends on multiple fields. Zod's .refine() and .superRefine() handle cross-field validation.

Confirm password

const schema = z.object({
  password: z.string().min(8, 'Min 8 characters'),
  confirmPassword: z.string(),
}).refine(
  (data) => data.password === data.confirmPassword,
  {
    message: 'Passwords do not match',
    path: ['confirmPassword'],  // Attach error to this field
  }
)
path is important: Without path, the error attaches to the root of the form (not any specific field). With path: ['confirmPassword'], the error shows up in errors.confirmPassword, so you can display it next to the right input.

Conditional validation

const schema = z.object({
  contactMethod: z.enum(['email', 'phone']),
  email: z.string().optional(),
  phone: z.string().optional(),
}).superRefine((data, ctx) => {
  // If they chose email, email is required
  if (data.contactMethod === 'email' && !data.email) {
    ctx.addIssue({
      code: z.ZodIssueCode.custom,
      message: 'Email is required',
      path: ['email'],
    })
  }

  // If they chose phone, phone is required
  if (data.contactMethod === 'phone' && !data.phone) {
    ctx.addIssue({
      code: z.ZodIssueCode.custom,
      message: 'Phone number is required',
      path: ['phone'],
    })
  }
})
Pydantic (model_validator)
class ContactForm(BaseModel):
    method: str
    email: str | None

    @model_validator(mode='after')
    def check(self):
        if self.method == 'email':
            assert self.email
        return self
Zod (superRefine)
schema.superRefine((data, ctx) => {
  if (data.method === 'email'
      && !data.email) {
    ctx.addIssue({
      code: z.ZodIssueCode.custom,
      message: 'Email required',
      path: ['email'],
    })
  }
})

Async Validation

Check server-side constraints: is this username taken? Is this email already registered? Zod supports async refinements.

const schema = z.object({
  username: z.string()
    .min(3, 'At least 3 characters')
    .refine(
      async (username) => {
        // Call your API to check availability
        const res = await fetch(
          `/api/check-username?u=${username}`
        )
        const { available } = await res.json()
        return available
      },
      { message: 'Username is already taken' }
    ),
  email: z.string().email(),
})
Performance consideration: Async validation runs on every validation trigger (submit, blur, change depending on your mode). For server checks, use mode: 'onBlur' so the API call only fires when the user leaves the field, not on every keystroke.

Error Display Patterns

Basic inline error

<div>
  <label>Email</label>
  <input
    {...register('email')}
    aria-invalid={errors.email ? 'true' : 'false'}
  />
  {errors.email && (
    <p role="alert" className="error">
      {errors.email.message}
    </p>
  )}
</div>

Reusable error component

import type { FieldError } from 'react-hook-form'

type Props = {
  error: FieldError | undefined
}

export const FieldError = ({ error }: Props) => {
  if (!error) return null
  return (
    <p role="alert" style={{ color: '#ef4444', fontSize: '0.875rem' }}>
      {error.message}
    </p>
  )
}

// Usage:
<input {...register('email')} />
<FieldError error={errors.email} />

Reusable form field wrapper

type FormFieldProps = {
  label: string
  error?: FieldError
  children: React.ReactNode
}

export const FormField = ({ label, error, children }: FormFieldProps) => (
  <div className={`field ${error ? 'field-error' : ''}`}>
    <label>{label}</label>
    {children}
    {error && <p className="error-msg">{error.message}</p>}
  </div>
)

// Usage:
<FormField label="Email" error={errors.email}>
  <input type="email" {...register('email')} />
</FormField>

Full Example: Registration Form

import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'

// ---- Schema (single source of truth) ----
const registrationSchema = z.object({
  username: z.string()
    .min(3, 'At least 3 characters')
    .max(20, 'Max 20 characters')
    .regex(/^[a-zA-Z0-9_]+$/, 'Letters, numbers, underscores only'),

  email: z.string()
    .min(1, 'Email is required')
    .email('Invalid email address'),

  password: z.string()
    .min(8, 'At least 8 characters')
    .regex(/[A-Z]/, 'Must contain an uppercase letter')
    .regex(/[0-9]/, 'Must contain a number'),

  confirmPassword: z.string(),

  age: z.coerce.number()
    .min(13, 'Must be at least 13')
    .max(120, 'Invalid age'),

  acceptTerms: z.literal(true, {
    errorMap: () => ({ message: 'You must accept the terms' }),
  }),
}).refine(
  (data) => data.password === data.confirmPassword,
  {
    message: 'Passwords do not match',
    path: ['confirmPassword'],
  }
)

// ---- Type (inferred from schema) ----
type RegistrationForm = z.infer<typeof registrationSchema>

// ---- Component ----
export const RegistrationPage = () => {
  const {
    register,
    handleSubmit,
    formState: { errors, isSubmitting },
  } = useForm<RegistrationForm>({
    resolver: zodResolver(registrationSchema),
    defaultValues: {
      username: '',
      email: '',
      password: '',
      confirmPassword: '',
      age: undefined as unknown as number,
      acceptTerms: false as unknown as true,
    },
  })

  const onSubmit = async (data: RegistrationForm) => {
    // data is fully validated and typed
    console.log('Registering:', data)
    await fetch('/api/register', {
      method: 'POST',
      body: JSON.stringify(data),
    })
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>

      <div>
        <label>Username</label>
        <input {...register('username')} />
        {errors.username && <p>{errors.username.message}</p>}
      </div>

      <div>
        <label>Email</label>
        <input type="email" {...register('email')} />
        {errors.email && <p>{errors.email.message}</p>}
      </div>

      <div>
        <label>Password</label>
        <input type="password" {...register('password')} />
        {errors.password && <p>{errors.password.message}</p>}
      </div>

      <div>
        <label>Confirm Password</label>
        <input type="password" {...register('confirmPassword')} />
        {errors.confirmPassword && <p>{errors.confirmPassword.message}</p>}
      </div>

      <div>
        <label>Age</label>
        <input type="number" {...register('age')} />
        {errors.age && <p>{errors.age.message}</p>}
      </div>

      <div>
        <label>
          <input type="checkbox" {...register('acceptTerms')} />
          I accept the terms and conditions
        </label>
        {errors.acceptTerms && <p>{errors.acceptTerms.message}</p>}
      </div>

      <button type="submit" disabled={isSubmitting}>
        {isSubmitting ? 'Creating account...' : 'Sign up'}
      </button>

    </form>
  )
}
