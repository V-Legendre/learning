10 — Form Basics

Controlled vs uncontrolled inputs, React Hook Form, register, handleSubmit, formState, and TypeScript integration.

Controlled vs Uncontrolled Inputs

In React, there are two ways to handle form inputs. This distinction doesn't exist in backend development — it's a DOM/browser concept. Understanding it is critical for choosing the right form library.

Controlled (React manages value)
const [email, setEmail] = useState('')

<input
  value={email}           {/* React controls the value */}
  onChange={(e) =>
    setEmail(e.target.value)
  }
/>

// Every keystroke:
// 1. onChange fires
// 2. setState called
// 3. Component re-renders
// 4. Input shows new value

Full control. You can validate, transform, or reject input on every keystroke. But every keystroke causes a re-render.

Uncontrolled (DOM manages value)
const inputRef = useRef<HTMLInputElement>(null)

<input
  ref={inputRef}         {/* DOM manages the value */}
  defaultValue=""
/>

// On submit:
const value = inputRef.current?.value
// No re-renders on keystroke!

DOM handles input natively. React reads the value only when needed (on submit). Zero re-renders while typing.

AspectControlledUncontrolled
Who manages valueReact state (useState)The DOM / browser
Re-renders on keystrokeYes (every character)No
When to read valueAnytime (it's in state)On submit (via ref)
Performance (large forms)Can be slow (many re-renders)Fast
Real-time validationEasy (validate in onChange)Need extra wiring
Python analogy: Think of controlled inputs like a Python @property setter that intercepts every assignment. Uncontrolled is like directly writing to an attribute and only validating when you need the final value.

Why React Hook Form?

Writing forms with raw React is tedious. For every field you need: state, onChange handler, validation logic, error messages, submit handling. For 10 fields, that's 10 useState calls and 10 handlers. React Hook Form (RHF) solves all of this.

What React Hook Form gives you
  • Performance: Uses uncontrolled inputs internally — minimal re-renders
  • Less boilerplate: One useForm() replaces all your useState/onChange pairs
  • Validation: Built-in rules or Zod integration (next page)
  • TypeScript support: Form values fully typed, field names autocompleted
  • Small bundle: ~9 KB gzipped, no dependencies
Without React Hook Form
const [email, setEmail] = useState('')
const [password, setPw] = useState('')
const [errors, setErrors] = useState({})
const [submitting, setSub] = useState(false)

const validate = () => {
  const errs: Record<string, string> = {}
  if (!email) errs.email = 'Required'
  if (!password) errs.password = 'Required'
  return errs
}

const onSubmit = async (e) => {
  e.preventDefault()
  const errs = validate()
  if (Object.keys(errs).length) {
    setErrors(errs); return
  }
  setSub(true)
  await login({ email, password })
  setSub(false)
}
// + all the JSX wiring...
With React Hook Form
const { register, handleSubmit,
        formState: { errors, isSubmitting }
      } = useForm<LoginForm>()

const onSubmit = async (
  data: LoginForm
) => {
  await login(data)
}

// That's it. register() handles
// all the input wiring.
Python analogy: React Hook Form is to React forms what WTForms is to Flask forms, or what Pydantic is to FastAPI request bodies. You declare the schema/type, and the library handles parsing, validation, and error reporting.

Core API

useForm<T>()

The main hook. Returns everything you need to manage a form. Type parameter T defines the shape of your form values.

import { useForm } from 'react-hook-form'

// Define the shape of your form data
type LoginForm = {
  email: string
  password: string
}

const {
  register,       // Connect an input to the form
  handleSubmit,   // Wraps your submit handler (validates first)
  formState,      // { errors, isSubmitting, isDirty, isValid, ... }
  watch,          // Subscribe to field value changes
  setValue,       // Programmatically set a field value
  reset,          // Reset form to default values
  getValues,      // Get current form values without subscribing
} = useForm<LoginForm>({
  defaultValues: {         // Initial values (optional)
    email: '',
    password: '',
  },
})

register("fieldName")

Returns props to spread onto an <input>. It sets up name, ref, onChange, and onBlur — everything the form needs to track that field.

// register returns: { name, ref, onChange, onBlur }
// You spread it onto the input:

<input {...register('email')} />

// Equivalent to writing:
<input
  name="email"
  ref={emailRef}
  onChange={handleEmailChange}
  onBlur={handleEmailBlur}
/>

// register with validation rules (built-in):
<input {...register('email', {
  required: 'Email is required',
  pattern: {
    value: /^[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}$/i,
    message: 'Invalid email address',
  },
})} />
The ... spread is essential. register() returns an object with multiple props. The spread syntax {...register('email')} puts all those props onto the input element. Without the spread, nothing works.

handleSubmit(onSubmit)

Wraps your submit handler. It validates the form first — your handler only runs if validation passes.

// handleSubmit validates, THEN calls your function
<form onSubmit={handleSubmit(onSubmit)}>

// Your onSubmit receives validated, typed data:
const onSubmit = async (data: LoginForm) => {
  // data.email is string (guaranteed by validation)
  // data.password is string
  console.log(data)
  await loginApi(data)
}

// Optional error handler (for failed validation):
const onError = (errors: FieldErrors<LoginForm>) => {
  console.log('Validation failed:', errors)
}

<form onSubmit={handleSubmit(onSubmit, onError)}>

formState

Object containing the current state of the form. Key properties:

PropertyTypeDescription
errorsFieldErrors<T>Validation errors, keyed by field name
isSubmittingbooleantrue while your async onSubmit runs
isDirtybooleantrue if any field has changed from default
isValidbooleantrue if all validation rules pass
dirtyFieldsRecord<string, boolean>Which specific fields have been modified
touchedFieldsRecord<string, boolean>Which fields the user has interacted with
submitCountnumberHow many times the form has been submitted

Complete Example: Login Form

import { useForm } from 'react-hook-form'

type LoginForm = {
  email: string
  password: string
}

export const LoginPage = () => {
  const {
    register,
    handleSubmit,
    formState: { errors, isSubmitting },
  } = useForm<LoginForm>({
    defaultValues: {
      email: '',
      password: '',
    },
  })

  const onSubmit = async (data: LoginForm) => {
    // data is fully typed and validated
    await fetch('/api/login', {
      method: 'POST',
      body: JSON.stringify(data),
    })
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>

      <div>
        <label htmlFor="email">Email</label>
        <input
          id="email"
          type="email"
          {...register('email', {
            required: 'Email is required',
          })}
        />
        {errors.email && (
          <span className="error">
            {errors.email.message}
          </span>
        )}
      </div>

      <div>
        <label htmlFor="password">Password</label>
        <input
          id="password"
          type="password"
          {...register('password', {
            required: 'Password is required',
            minLength: {
              value: 8,
              message: 'At least 8 characters',
            },
          })}
        />
        {errors.password && (
          <span className="error">
            {errors.password.message}
          </span>
        )}
      </div>

      <button
        type="submit"
        disabled={isSubmitting}
      >
        {isSubmitting ? 'Logging in...' : 'Log in'}
      </button>

    </form>
  )
}

TypeScript Integration

When you pass a type parameter to useForm<LoginForm>(), everything becomes type-safe:

What the type parameter gives you
  • register('email') — TypeScript autocompletes field names, errors on typos
  • register('emal') — TypeScript ERROR (field doesn't exist in LoginForm)
  • errors.email?.message — Typed error access
  • onSubmit(data: LoginForm) — Submit handler receives typed, validated data
  • watch('email') — Returns string (inferred from the type)
// TypeScript catches mistakes at compile time:

type SignupForm = {
  username: string
  email: string
  age: number
}

const { register } = useForm<SignupForm>()

register('username')  // OK
register('email')     // OK
register('age')       // OK
register('name')      // ERROR: 'name' is not in SignupForm

Different Input Types

React Hook Form works with all standard HTML inputs. Here's how each type is registered:

type ProfileForm = {
  name: string
  bio: string
  role: 'developer' | 'designer' | 'manager'
  newsletter: boolean
}

// Text input
<input type="text" {...register('name')} />

// Textarea
<textarea {...register('bio')} />

// Select dropdown
<select {...register('role')}>
  <option value="developer">Developer</option>
  <option value="designer">Designer</option>
  <option value="manager">Manager</option>
</select>

// Checkbox
<input type="checkbox" {...register('newsletter')} />

Validation Modes

By default, validation runs on submit. You can change when validation triggers:

const form = useForm<LoginForm>({
  mode: 'onSubmit',      // Default: validate on submit
  // mode: 'onBlur',      // Validate when field loses focus
  // mode: 'onChange',    // Validate on every keystroke (expensive)
  // mode: 'onTouched',  // Validate on blur, then on change after first error
  // mode: 'all',        // All of the above
})
Recommendation: Start with 'onSubmit' (default). If you want inline validation feedback, use 'onBlur' — it validates when the user tabs away from a field, which feels natural. Avoid 'onChange' unless you have a specific real-time validation need, as it triggers on every keystroke.
