11 — Zustand

Minimal, type-safe client state management without the boilerplate.

Why Zustand?

Zustand (German for "state") is a tiny (~1KB) state management library that beats the alternatives in simplicity and performance.

Feature Redux Toolkit Context API Zustand
Bundle size ~11KB 0 (built-in) ~1KB
Boilerplate High (slices, reducers, actions) Medium (Provider, context) Minimal
TypeScript Good Built-in Excellent
Providers needed Yes (store provider) Yes (context provider) No
Re-render control Selectors None (all consumers re-render) Selectors (built-in)
DevTools Built-in React DevTools only Via middleware
Learning curve Steep Low Very low
Python Analogy

Zustand is like a typed singleton module with subscription callbacks. Think of a Python module-level state with an observer pattern:

# Python: module-level state
class Store:
    def __init__(self):
        self.count = 0
        self._listeners = []

    def increment(self):
        self.count += 1
        for cb in self._listeners:
            cb(self)

    def subscribe(self, cb):
        self._listeners.append(cb)

store = Store()  # singleton
Zustand equivalent
// Zustand: same idea, way less code
import { create } from 'zustand'

interface CounterState {
  count: number
  increment: () => void
}

const useCounterStore = create<CounterState>()(
  (set) => ({
    count: 0,
    increment: () =>
      set((state) => ({
        count: state.count + 1
      })),
  })
)

Creating a Store

A Zustand store is created with create. It returns a hook (by convention prefixed with use). The function receives set and get to update and read state.

Complete typed store example
import { create } from 'zustand'

// 1. Define the state type
interface TodoState {
  // State
  todos: Array<{ id: string; text: string; done: boolean }>
  filter: 'all' | 'active' | 'done'

  // Actions
  addTodo: (text: string) => void
  toggleTodo: (id: string) => void
  removeTodo: (id: string) => void
  setFilter: (filter: TodoState['filter']) => void
}

// 2. Create the store
const useTodoStore = create<TodoState>()((set) => ({
  todos: [],
  filter: 'all',

  addTodo: (text) =>
    set((state) => ({
      todos: [...state.todos, {
        id: crypto.randomUUID(),
        text,
        done: false,
      }],
    })),

  toggleTodo: (id) =>
    set((state) => ({
      todos: state.todos.map((t) =>
        t.id === id ? { ...t, done: !t.done } : t
      ),
    })),

  removeTodo: (id) =>
    set((state) => ({
      todos: state.todos.filter((t) => t.id !== id),
    })),

  setFilter: (filter) => set({ filter }),
}))
Note: set does a shallow merge by default. You only need to return the properties you want to change. set({ filter: 'done' }) merges into existing state, preserving todos.

Using a Store in Components

Without selector (BAD)
const TodoCount = () => {
  // Subscribes to ENTIRE store
  // Re-renders on ANY state change
  const state = useTodoStore()
  return <p>{state.todos.length} todos</p>
}
With selector (GOOD)
const TodoCount = () => {
  // Only subscribes to todos
  // Only re-renders when todos change
  const todos = useTodoStore(
    (s) => s.todos
  )
  return <p>{todos.length} todos</p>
}
Always use selectors. const state = useStore() with no argument subscribes to the entire store. Every update to any property will re-render that component. Select only what you need.

Selecting Multiple Values

// Option 1: Multiple selectors (preferred for unrelated values)
const TodoList = () => {
  const todos = useTodoStore((s) => s.todos)
  const filter = useTodoStore((s) => s.filter)
  const toggleTodo = useTodoStore((s) => s.toggleTodo)
  // ...
}

// Option 2: useShallow for object selection
import { useShallow } from 'zustand/react/shallow'

const TodoList = () => {
  const { todos, filter, toggleTodo } = useTodoStore(
    useShallow((s) => ({
      todos: s.todos,
      filter: s.filter,
      toggleTodo: s.toggleTodo,
    }))
  )
}
Tip: Actions (functions) never change, so selecting them doesn't cause re-renders. You can safely select them without useShallow. The re-render concern is about data values.

Accessing the Store Outside React

Unlike Context, Zustand stores work outside of React components. This is useful for utility functions, event handlers, and testing.

// Read current state (snapshot, not reactive)
const currentTodos = useTodoStore.getState().todos

// Call actions directly
useTodoStore.getState().addTodo('Buy milk')

// Subscribe to changes (returns unsubscribe function)
const unsub = useTodoStore.subscribe((state) => {
  console.log('State changed:', state)
})

Middleware

Zustand middleware wraps the store to add functionality. They compose with function nesting.

persist — Save to localStorage

import { create } from 'zustand'
import { persist } from 'zustand/middleware'

interface ThemeState {
  theme: 'light' | 'dark'
  toggle: () => void
}

const useThemeStore = create<ThemeState>()(
  persist(
    (set) => ({
      theme: 'dark',
      toggle: () =>
        set((s) => ({
          theme: s.theme === 'dark' ? 'light' : 'dark',
        })),
    }),
    {
      name: 'theme-storage', // localStorage key
    }
  )
)
Note: The persist middleware automatically serializes to JSON and hydrates on page load. You can configure which parts to persist with partialize, and use custom storage backends.

devtools — Redux DevTools Integration

import { devtools } from 'zustand/middleware'

const useStore = create<State>()(
  devtools(
    (set) => ({
      count: 0,
      increment: () =>
        set(
          (s) => ({ count: s.count + 1 }),
          false,     // replace (false = merge)
          'increment' // action name in DevTools
        ),
    }),
    { name: 'CounterStore' }
  )
)

immer — Mutable Update Syntax

Without immer (immutable)
toggleTodo: (id) =>
  set((state) => ({
    todos: state.todos.map((t) =>
      t.id === id
        ? { ...t, done: !t.done }
        : t
    ),
  }))
With immer (mutable syntax)
import { immer } from 'zustand/middleware/immer'

toggleTodo: (id) =>
  set((state) => {
    const todo = state.todos.find(
      (t) => t.id === id
    )
    if (todo) todo.done = !todo.done
  })
Tip: Immer uses proxies under the hood — your mutations are actually producing new immutable objects. This is the same Immer library Redux Toolkit uses.

Combining Middleware

// Middleware composes from inside out
const useStore = create<State>()(
  devtools(
    persist(
      immer(
        (set) => ({
          // your state and actions
        })
      ),
      { name: 'my-store' }
    ),
    { name: 'MyStore' }
  )
)

Multiple Stores

Prefer multiple small stores over one big store. Organize by domain, not by type.

src/
  stores/
    useAuthStore.ts   // user, login, logout
    useCartStore.ts   // items, addItem, removeItem
    useUIStore.ts     // sidebar, theme, modals
    useNotificationStore.ts // toasts, alerts
One mega store
// Everything in one store
// Hard to maintain, test, reason about
const useStore = create((set) => ({
  user: null,
  cartItems: [],
  theme: 'dark',
  sidebarOpen: false,
  notifications: [],
  // 50 more properties...
  // 30 more actions...
}))
Small, focused stores
// Each store has a single responsibility
const useAuthStore = create(() => ({
  user: null,
  login: ...,
  logout: ...,
}))

const useCartStore = create(() => ({
  items: [],
  addItem: ...,
  removeItem: ...,
}))

Testing Zustand Stores

Since stores are just functions, you can test them outside of React. No rendering, no providers, no hooks.

import { describe, it, expect, beforeEach } from 'vitest'
import { useTodoStore } from './useTodoStore'

describe('useTodoStore', () => {
  // Reset store between tests
  beforeEach(() => {
    useTodoStore.setState({ todos: [], filter: 'all' })
  })

  it('adds a todo', () => {
    useTodoStore.getState().addTodo('Buy milk')
    const { todos } = useTodoStore.getState()

    expect(todos).toHaveLength(1)
    expect(todos[0].text).toBe('Buy milk')
    expect(todos[0].done).toBe(false)
  })

  it('toggles a todo', () => {
    useTodoStore.getState().addTodo('Buy milk')
    const id = useTodoStore.getState().todos[0].id

    useTodoStore.getState().toggleTodo(id)
    expect(useTodoStore.getState().todos[0].done).toBe(true)
  })
})
Key advantage over Context: Context-based state requires rendering components with providers to test. Zustand stores are plain functions — test them like you'd test a Python class in pytest.

Full Example: Shopping Cart Store

useCartStore.ts — complete, production-style store
import { create } from 'zustand'
import { persist, devtools } from 'zustand/middleware'

interface CartItem {
  id: string
  name: string
  price: number
  quantity: number
}

interface CartState {
  items: CartItem[]
  addItem: (item: Omit<CartItem, 'quantity'>) => void
  removeItem: (id: string) => void
  updateQuantity: (id: string, quantity: number) => void
  clearCart: () => void
  totalItems: () => number
  totalPrice: () => number
}

export const useCartStore = create<CartState>()(
  devtools(
    persist(
      (set, get) => ({
        items: [],

        addItem: (item) =>
          set((state) => {
            const existing = state.items.find((i) => i.id === item.id)
            if (existing) {
              return {
                items: state.items.map((i) =>
                  i.id === item.id
                    ? { ...i, quantity: i.quantity + 1 }
                    : i
                ),
              }
            }
            return { items: [...state.items, { ...item, quantity: 1 }] }
          }),

        removeItem: (id) =>
          set((state) => ({
            items: state.items.filter((i) => i.id !== id),
          })),

        updateQuantity: (id, quantity) =>
          set((state) => ({
            items: quantity <= 0
              ? state.items.filter((i) => i.id !== id)
              : state.items.map((i) =>
                  i.id === id ? { ...i, quantity } : i
                ),
          })),

        clearCart: () => set({ items: [] }),

        // Computed values using get()
        totalItems: () =>
          get().items.reduce((sum, i) => sum + i.quantity, 0),

        totalPrice: () =>
          get().items.reduce(
            (sum, i) => sum + i.price * i.quantity, 0
          ),
      }),
      { name: 'cart-storage' }
    ),
    { name: 'CartStore' }
  )
)

Using the Cart Store

const CartIcon = () => {
  const totalItems = useCartStore((s) => s.totalItems())
  return <span>Cart ({totalItems})</span>
}

const CartTotal = () => {
  const totalPrice = useCartStore((s) => s.totalPrice())
  return <p>Total: ${totalPrice.toFixed(2)}</p>
}

const AddToCartButton = ({ product }: { product: Product }) => {
  const addItem = useCartStore((s) => s.addItem)
  return (
    <button onClick={() => addItem(product)}>
      Add to Cart
    </button>
  )
}

Summary

Zustand in 30 seconds
  • create<Type>()((set, get) => ({ ... })) — define state + actions
  • useStore((s) => s.value) — always use selectors in components
  • useStore.getState() — access outside React
  • persist — save to localStorage
  • devtools — Redux DevTools integration
  • immer — mutable update syntax
  • Multiple small stores > one mega store
  • Test stores as plain functions — no React needed
