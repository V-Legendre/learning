04 — Project Setup

Creating a React + TypeScript project with Vite and pnpm.

What Is Vite?

Vite (French for "fast") is a build tool and dev server for modern frontend projects. It replaced older tools like webpack and Create React App (CRA), which are now effectively deprecated.

What Vite does
  • Dev server — serves your files with instant Hot Module Replacement (HMR)
  • Bundler — compiles TypeScript, bundles for production (uses Rollup under the hood)
  • Config — minimal config needed, sensible defaults
Python analogy

uvicorn --reload watches for file changes and restarts your FastAPI server.

pyproject.toml defines dependencies and build config.

pip install / poetry install manages packages.

Vite equivalent

vite dev watches for file changes and hot-reloads your components without a full page refresh.

vite.config.ts defines build config.

pnpm install manages packages via package.json.

Why Vite over webpack? Webpack bundles your entire app before serving anything (slow). Vite serves files as native ES modules — the browser loads only what it needs. Dev startup is nearly instant regardless of project size.

Key files explained

index.html — The single page

This is the only HTML file in a React SPA (Single-Page Application). The entire app lives inside one <div id="root">.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Vite + React + TS</title>
</head>
<body>
  <div id="root"></div>       <!-- React mounts here -->
  <script type="module" src="/src/main.tsx"></script>
</body>
</html>

Notice the <script type="module"> — Vite uses native ES modules. No bundling in dev mode.

src/main.tsx — Entry point

This file bridges the DOM and React. It finds the #root div and mounts your app into it.

import React from 'react'
import ReactDOM from 'react-dom/client'
import { App } from './App'
import './index.css'

ReactDOM.createRoot(
  document.getElementById('root')!   // ! = non-null assertion (we know it exists)
).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)
React.StrictMode is a development-only wrapper that helps find bugs. It intentionally double-renders components to catch side effects, warns about deprecated patterns, and adds extra checks. Has zero effect in production builds.
src/App.tsx — Root component

Your top-level component. Everything you build will be composed under this.

const App = () => {
  return (
    <div>
      <h1>Hello, React!</h1>
    </div>
  )
}

export { App }
vite.config.ts — Build configuration
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],         // Enables JSX transform, Fast Refresh
})

Minimal config. The React plugin handles JSX compilation and HMR. You can add more plugins for things like path aliases, CSS modules, etc.

package.json — Dependencies & scripts
{
  "name": "my-app",
  "private": true,
  "type": "module",
  "scripts": {
    "dev": "vite",                 // Start dev server
    "build": "tsc -b && vite build",  // Type-check then bundle
    "preview": "vite preview"       // Serve production build locally
  },
  "dependencies": {
    "react": "^19.0.0",
    "react-dom": "^19.0.0"
  },
  "devDependencies": {
    "@types/react": "^19.0.0",       // TypeScript types for React
    "@types/react-dom": "^19.0.0",   // TypeScript types for ReactDOM
    "@vitejs/plugin-react": "^4.x",  // Vite's React plugin
    "typescript": "~5.x",            // TypeScript compiler
    "vite": "^6.x"                   // Build tool
  }
}

Hot Module Replacement (HMR)

HMR means when you save a file, the browser updates only the changed module without a full page reload. Your component state is preserved.

1
Save file
You edit App.tsx and save
->
2
Vite detects change
File watcher triggers (via WebSocket)
->
3
Compiles module
Only the changed file, not the whole app
->
4
Browser swaps module
React re-renders component, state preserved
Without HMR (old tools)
  • Full page reload on every change
  • All component state is lost
  • You have to navigate back to the page you were testing
  • Slow for large projects
With Vite HMR
  • Sub-second updates
  • Component state preserved (form inputs, scroll position)
  • Only the changed module reloads
  • Speed is independent of project size
How Vite achieves this: In dev mode, Vite serves each file as a native ES module (via <script type="module">). The browser's native module system handles dependency loading. No bundling step in dev. This is why Vite starts instantly even on huge projects.

TypeScript Strict Mode

The Vite template enables "strict": true in tsconfig.json. This turns on a collection of stricter type-checking flags that catch bugs at compile time.

What "strict" enables
FlagWhat it doesWhy it matters
strictNullChecks null and undefined are not assignable to other types Forces you to handle the "no value" case
noImplicitAny Error if TS can't infer a type and you didn't annotate No silent any types sneaking in
strictFunctionTypes Stricter checking of function parameter types Prevents unsound callback typing
strictPropertyInitialization Class properties must be initialized Prevents undefined property access
noImplicitThis Error on this with implicit any type Catches this context bugs
alwaysStrict Emit "use strict" in every file Enables JS strict mode
Python analogy: Strict mode is like running mypy --strict on your Python code. Without it, TypeScript is more lenient — like running mypy in its default mode where many things are untyped. Always use strict mode for new projects.

From zero to running app
$ pnpm create vite@latest my-app --template react-ts
$ cd my-app
$ pnpm install
$ pnpm dev

-> Local: http://localhost:5173/
Common mistake: Running npx create-react-app — that's the old way (CRA). It's unmaintained, slow, and uses webpack. Always use pnpm create vite@latest for new React projects.
