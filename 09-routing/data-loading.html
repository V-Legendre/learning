09 — Data Loading

Route loaders, TanStack Query integration, pending UI, error handling, and preloading for instant navigation.

The Problem: Fetch-in-Component

The naive approach is to fetch data inside useEffect when the component mounts. This creates two major problems:

Problem 1: Loading spinners everywhere

Every component shows its own spinner independently. The user sees a skeleton page fill in piece by piece. Jarring UX.

function UserProfile() {
  const [user, setUser] = useState(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    fetchUser(id).then(setUser)
      .finally(() => setLoading(false))
  }, [id])

  if (loading) return <Spinner />
  // ...render user
}
Problem 2: Request waterfalls

Parent loads data, THEN renders child, THEN child loads its data. Sequential requests that could have been parallel.

// Timeline:
// 1. Navigate to /users/42
// 2. UserLayout mounts → fetches user
// 3. ...500ms later, user loads...
// 4. UserPosts mounts → fetches posts
// 5. ...500ms later, posts load...
// Total: 1000ms (sequential!)
//
// With loaders: both fetch in parallel
// Total: 500ms
Without loaders
Navigate → Render → Fetch → Spinner → Render children → Fetch → Spinner
→
With loaders
Navigate → Fetch all data → Render everything

Route Loaders

A loader is a function that runs before the route component renders. It fetches all the data the route needs, so by the time the component mounts, everything is ready. No useEffect, no loading state, no spinners in the component itself.

FastAPI dependency injection
# Data resolved BEFORE endpoint runs
async def get_user(
    user_id: int,
    db: Session = Depends(get_db)
) -> User:
    return db.query(User).get(user_id)

@app.get("/users/{user_id}")
async def user_detail(
    user: User = Depends(get_user)
):
    # user is already loaded!
    return user
TanStack Router loader
// Data resolved BEFORE component renders
export const Route = createFileRoute(
  '/users/$userId'
)({
  loader: async ({ params }) => {
    const user = await fetchUser(params.userId)
    return { user }
  },
  component: UserDetail,
})

function UserDetail() {
  // user is already loaded!
  const { user } = Route.useLoaderData()
  return <h1>{user.name}</h1>
}

Loader function signature

export const Route = createFileRoute('/users/$userId')({
  loader: async ({
    params,        // { userId: string } — from the URL
    search,        // validated search params (if validateSearch is set)
    context,       // shared context (auth, queryClient, etc.)
    abortController, // cancel signal if navigation changes
  }) => {
    const user = await fetch(`/api/users/${params.userId}`)
      .then(r => r.json())

    return { user }  // Returned data is available via useLoaderData()
  },
})

TanStack Query Integration

Loaders alone work, but they re-fetch on every navigation. Combining loaders with TanStack Query gives you: caching, background refetching, deduplication, and stale-while-revalidate. The loader just ensures the data exists in the cache.

The pattern: queryOptions + ensureQueryData

// src/lib/queries/users.ts — Define query options once, reuse everywhere
import { queryOptions } from '@tanstack/react-query'

export const userQueryOptions = (userId: string) =>
  queryOptions({
    queryKey: ['users', userId],
    queryFn: () => fetch(`/api/users/${userId}`).then(r => r.json()),
  })

export const usersListQueryOptions = (page: number) =>
  queryOptions({
    queryKey: ['users', 'list', page],
    queryFn: () => fetch(`/api/users?page=${page}`).then(r => r.json()),
  })
// src/routes/users/$userId.tsx — Use in the loader
import { createFileRoute } from '@tanstack/react-router'
import { useSuspenseQuery } from '@tanstack/react-query'
import { userQueryOptions } from '../../lib/queries/users'

export const Route = createFileRoute('/users/$userId')({
  loader: ({ context, params }) => {
    // ensureQueryData: fetch if not cached, return cache if fresh
    context.queryClient.ensureQueryData(
      userQueryOptions(params.userId)
    )
  },
  component: UserDetail,
})

function UserDetail() {
  const { userId } = Route.useParams()

  // Data is guaranteed to be in cache (loader ensured it)
  // useSuspenseQuery reads from cache, no loading state needed
  const { data: user } = useSuspenseQuery(
    userQueryOptions(userId)
  )

  return <h1>{user.name}</h1>
}
How to pass queryClient via context
// src/main.tsx
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'

const queryClient = new QueryClient()

const router = createRouter({
  routeTree,
  context: { queryClient },  // Available in every loader
})

// In __root.tsx, wrap with QueryClientProvider
export const Route = createRootRouteWithContext<{
  queryClient: QueryClient
}>()({
  component: () => (
    <QueryClientProvider client={queryClient}>
      <Outlet />
    </QueryClientProvider>
  ),
})
Why both loader + useSuspenseQuery? The loader fires on navigation, ensuring data is cached. useSuspenseQuery reads from that cache and also handles background refetching when data becomes stale. Loader handles the "before render" part, Query handles the "stay fresh" part.

Pending UI

While the loader fetches data, the user sees... what? TanStack Router gives you several options for pending/loading states.

Option 1: pendingComponent

export const Route = createFileRoute('/users/$userId')({
  loader: async ({ params }) => {
    return { user: await fetchUser(params.userId) }
  },
  pendingComponent: () => <div>Loading user...</div>,
  component: UserDetail,
})

Option 2: pendingMinMs and pendingMs

const router = createRouter({
  routeTree,
  defaultPendingMs: 200,     // Wait 200ms before showing pending UI
  defaultPendingMinMs: 300,  // Show pending UI for at least 300ms (no flash)
})
Why pendingMs? If data loads in 50ms, you don't want a loading spinner that appears and disappears instantly (flash). The 200ms delay means: if the loader resolves within 200ms, no spinner is shown at all. If it takes longer, the spinner appears and stays for at least 300ms (to avoid a jarring flash).

Option 3: Global loading indicator

// In __root.tsx — show a top bar while any route is loading
import { useRouterState } from '@tanstack/react-router'

function RootComponent() {
  const isLoading = useRouterState({
    select: (s) => s.isLoading,
  })

  return (
    <div>
      {isLoading && <div className="loading-bar" />}
      <Outlet />
    </div>
  )
}

Error Handling in Loaders

When a loader throws, the route's errorComponent renders instead of the regular component. This is similar to a try/except around your Flask view function.

export const Route = createFileRoute('/users/$userId')({
  loader: async ({ params }) => {
    const res = await fetch(`/api/users/${params.userId}`)

    if (res.status === 404) {
      throw notFound()  // Shows notFoundComponent
    }

    if (!res.ok) {
      throw new Error(`Failed to fetch user: ${res.status}`)
    }

    return { user: await res.json() }
  },

  errorComponent: ({ error }) => (
    <div>
      <h2>Something went wrong</h2>
      <p>{error.message}</p>
      <Link to="/users">Back to users</Link>
    </div>
  ),

  notFoundComponent: () => (
    <div>
      <h2>User not found</h2>
      <Link to="/users">Browse all users</Link>
    </div>
  ),

  component: UserDetail,
})

Redirect on auth failure

import { redirect } from '@tanstack/react-router'

export const Route = createFileRoute('/dashboard')({
  loader: async ({ context }) => {
    const res = await fetch('/api/me')

    if (res.status === 401) {
      // Redirect to login — throw, don't return
      throw redirect({
        to: '/login',
        search: { redirect: '/dashboard' },
      })
    }

    return { user: await res.json() }
  },
})

Preloading

Preloading fetches route data before the user actually navigates — typically on link hover or link visibility. When the user finally clicks, the page appears instantly because the data is already cached.

// Enable preloading globally
const router = createRouter({
  routeTree,
  defaultPreload: 'intent',       // Preload on hover/focus
  defaultPreloadStaleTime: 5000,  // Cache preloaded data for 5s
})

// Or per-link:
<Link
  to="/users/$userId"
  params={{ userId: '42' }}
  preload="intent"        {/* hover/focus triggers preload */}
>
  View User
</Link>

// preload options:
// "intent"   — preload on hover/focus (most common)
// "viewport" — preload when link is visible
// "render"   — preload immediately when link renders
// false      — disable preloading

Data Loading Patterns Compared

PatternWhen data loadsRe-fetchingCachingBest for
useEffect + useState After component mounts Manual None Never (avoid this)
Loader only Before component renders On every navigation None Simple, small apps
Loader + TanStack Query Before render, cached Automatic (stale-while-revalidate) Full Most apps (recommended)
Python analogy: The loader pattern is like FastAPI's dependency injection: data is resolved before your handler (component) runs. ensureQueryData is like a caching layer in front of your database — if the data is fresh in Redis, skip the DB query; otherwise, fetch and cache it.

Full Example: Posts with Loader + Query

// src/lib/queries/posts.ts
import { queryOptions } from '@tanstack/react-query'

export const postsQueryOptions = (page: number) =>
  queryOptions({
    queryKey: ['posts', { page }],
    queryFn: async () => {
      const res = await fetch(`/api/posts?page=${page}`)
      if (!res.ok) throw new Error('Failed to fetch posts')
      return res.json() as Promise<Post[]>
    },
  })

export const postQueryOptions = (postId: string) =>
  queryOptions({
    queryKey: ['posts', postId],
    queryFn: async () => {
      const res = await fetch(`/api/posts/${postId}`)
      if (!res.ok) throw new Error('Post not found')
      return res.json() as Promise<Post>
    },
  })
// src/routes/posts/$postId.tsx
import { createFileRoute } from '@tanstack/react-router'
import { useSuspenseQuery } from '@tanstack/react-query'
import { postQueryOptions } from '../../lib/queries/posts'

export const Route = createFileRoute('/posts/$postId')({
  loader: ({ context, params }) =>
    context.queryClient.ensureQueryData(
      postQueryOptions(params.postId)
    ),

  pendingComponent: () => <div>Loading post...</div>,

  errorComponent: ({ error }) => (
    <div>
      <h2>Error loading post</h2>
      <p>{error.message}</p>
    </div>
  ),

  component: PostDetail,
})

function PostDetail() {
  const { postId } = Route.useParams()
  const { data: post } = useSuspenseQuery(postQueryOptions(postId))

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.body}</p>
      <small>By {post.author} on {post.date}</small>
    </article>
  )
}
