09 — Auth & Protected Routes

Route guards with beforeLoad, auth context, layout-level protection, redirects, and role-based access.

The Pattern

Many routes in a real app require the user to be logged in. If they're not, redirect them to /login. After they log in, send them back to where they were trying to go. This is the same pattern as Flask's @login_required decorator.

Flask / FastAPI
# Flask: decorator-based guard
@app.route("/dashboard")
@login_required
def dashboard():
    return render_template("dash.html")

# FastAPI: dependency injection
@app.get("/dashboard")
async def dashboard(
    user: User = Depends(get_current_user)
):
    return {"user": user}
TanStack Router
// beforeLoad: runs before the loader
export const Route = createFileRoute(
  '/dashboard'
)({
  beforeLoad: ({ context }) => {
    if (!context.auth.isAuthenticated) {
      throw redirect({
        to: '/login',
      })
    }
  },
  component: Dashboard,
})

The beforeLoad Guard

beforeLoad runs before the route's loader and component. It's the first thing that executes when navigating to a route. If it throws a redirect, the navigation is cancelled and the user is sent elsewhere.

1
Navigation starts
User clicks link or enters URL
→
2
beforeLoad
Auth check here
→
3
loader
Fetch data
→
4
component
Render UI

Complete beforeLoad example

import { createFileRoute, redirect } from '@tanstack/react-router'

export const Route = createFileRoute('/dashboard')({
  beforeLoad: async ({ context, location }) => {
    // context.auth comes from router context (set up in main.tsx)
    if (!context.auth.isAuthenticated) {
      throw redirect({
        to: '/login',
        search: {
          // Save where they were trying to go
          redirect: location.href,
        },
      })
    }
  },
  loader: async ({ context }) => {
    // If we get here, user is authenticated
    return { dashboardData: await fetchDashboard() }
  },
  component: Dashboard,
})

function Dashboard() {
  const { dashboardData } = Route.useLoaderData()
  return <h1>Welcome to your dashboard</h1>
}
throw, don't return: You must throw redirect(), not return redirect(). The throw is what short-circuits the route loading pipeline. If you return it, it's just a regular object and the route will try to render normally.

Setting Up Auth Context

The router needs access to auth state. You pass it via router context, which makes it available in every beforeLoad and loader.

Step 1: Define the auth store

// src/lib/auth.ts
import { create } from 'zustand'

type User = {
  id: string
  email: string
  role: 'admin' | 'user'
}

type AuthState = {
  user: User | null
  isAuthenticated: boolean
  login: (email: string, password: string) => Promise<void>
  logout: () => void
}

export const useAuthStore = create<AuthState>((set) => ({
  user: null,
  isAuthenticated: false,

  login: async (email, password) => {
    const res = await fetch('/api/login', {
      method: 'POST',
      body: JSON.stringify({ email, password }),
    })
    const user = await res.json()
    set({ user, isAuthenticated: true })
  },

  logout: () => {
    set({ user: null, isAuthenticated: false })
  },
}))

Step 2: Pass auth to router context

// src/main.tsx
import { createRouter, RouterProvider } from '@tanstack/react-router'
import { routeTree } from './routeTree.gen'
import { useAuthStore } from './lib/auth'

const router = createRouter({
  routeTree,
  context: {
    auth: undefined!,  // Will be set at render time
  },
})

function App() {
  const auth = useAuthStore()

  return (
    <RouterProvider
      router={router}
      context={{ auth }}  {/* Fresh auth state on every render */}
    />
  )
}

Step 3: Type the context in __root.tsx

// src/routes/__root.tsx
import { createRootRouteWithContext, Outlet } from '@tanstack/react-router'
import type { AuthState } from '../lib/auth'

type RouterContext = {
  auth: AuthState
}

export const Route = createRootRouteWithContext<RouterContext>()({
  component: () => (
    <div>
      <NavBar />
      <Outlet />
    </div>
  ),
})
Now every route has typed access to context.auth in both beforeLoad and loader. TypeScript knows the exact shape of your auth state.

Layout-Level Protection

Instead of adding beforeLoad to every protected route, put the guard on a pathless layout. All child routes inherit the protection.

// src/routes/_authenticated.tsx — Protects ALL children
import { createFileRoute, redirect, Outlet } from '@tanstack/react-router'

export const Route = createFileRoute('/_authenticated')({
  beforeLoad: async ({ context, location }) => {
    if (!context.auth.isAuthenticated) {
      throw redirect({
        to: '/login',
        search: { redirect: location.href },
      })
    }
  },
  component: () => <Outlet />,  {/* Just passes through */}
})
src/routes/
  __root.tsx
  index.tsx ← Public (/)
  login.tsx ← Public (/login)
  about.tsx ← Public (/about)
  _authenticated.tsx ← Guard: beforeLoad checks auth
  _authenticated/
    dashboard.tsx ← Protected (/dashboard)
    settings.tsx ← Protected (/settings)
    profile.tsx ← Protected (/profile)
Key advantage: The _authenticated prefix means it doesn't add to the URL. /dashboard is the URL, not /authenticated/dashboard. The underscore prefix makes it a pathless layout — it only groups routes for shared behavior.

Redirect After Login

When a user gets bounced to /login, save where they wanted to go. After successful login, send them back.

// src/routes/login.tsx
import { createFileRoute, useNavigate } from '@tanstack/react-router'
import { z } from 'zod'
import { useAuthStore } from '../lib/auth'

const loginSearchSchema = z.object({
  redirect: z.string().optional(),
})

export const Route = createFileRoute('/login')({
  validateSearch: loginSearchSchema,
  component: LoginPage,
})

function LoginPage() {
  const navigate = useNavigate()
  const { redirect: redirectUrl } = Route.useSearch()
  const login = useAuthStore((s) => s.login)

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault()

    await login(email, password)

    // Redirect to original destination, or dashboard as fallback
    navigate({
      to: redirectUrl ?? '/dashboard',
      replace: true,  // Don't put login page in history
    })
  }

  return (
    <form onSubmit={handleSubmit}>
      <h1>Login</h1>
      {redirectUrl && (
        <p className="notice">
          Please log in to continue.
        </p>
      )}
      {/* email & password inputs */}
      <button type="submit">Log in</button>
    </form>
  )
}

Route-Level Permissions (Roles)

Beyond just "logged in or not," you often need role-based access. Admins can access /admin, regular users can't.

// src/routes/_authenticated/_admin.tsx
// Nested pathless layout: requires auth AND admin role
import { createFileRoute, redirect, Outlet } from '@tanstack/react-router'

export const Route = createFileRoute('/_authenticated/_admin')({
  beforeLoad: ({ context }) => {
    // Auth is already checked by parent _authenticated layout
    // Here we just check the role
    if (context.auth.user?.role !== 'admin') {
      throw redirect({ to: '/dashboard' })
    }
  },
  component: () => <Outlet />,
})
src/routes/
  _authenticated.tsx ← Auth guard
  _authenticated/
    dashboard.tsx ← Any logged-in user (/dashboard)
    _admin.tsx ← Admin role guard
    _admin/
      admin-panel.tsx ← Admins only (/admin-panel)
      user-management.tsx ← Admins only (/user-management)

Reusable guard helper

If you have many role checks, create a helper function to keep routes DRY:

// src/lib/route-guards.ts
import { redirect } from '@tanstack/react-router'
import type { RouterContext } from '../routes/__root'

export const requireAuth = ({ context, location }: {
  context: RouterContext
  location: { href: string }
}) => {
  if (!context.auth.isAuthenticated) {
    throw redirect({
      to: '/login',
      search: { redirect: location.href },
    })
  }
}

export const requireRole = (
  role: 'admin' | 'user',
  { context }: { context: RouterContext }
) => {
  if (context.auth.user?.role !== role) {
    throw redirect({ to: '/dashboard' })
  }
}

// Usage in a route:
export const Route = createFileRoute('/admin')({
  beforeLoad: (opts) => {
    requireAuth(opts)
    requireRole('admin', opts)
  },
  component: AdminPanel,
})

Conditional Navigation UI

Show or hide navigation items based on auth state. The nav bar should reflect what the user can access.

// In __root.tsx or a NavBar component
import { useAuthStore } from '../lib/auth'

function NavBar() {
  const { isAuthenticated, user, logout } = useAuthStore()
  const navigate = useNavigate()

  return (
    <nav>
      <Link to="/">Home</Link>

      {isAuthenticated ? (
        <>
          <Link to="/dashboard">Dashboard</Link>
          {user?.role === 'admin' && (
            <Link to="/admin-panel">Admin</Link>
          )}
          <button onClick={() => {
            logout()
            navigate({ to: '/' })
          }}>
            Logout
          </button>
        </>
      ) : (
        <Link to="/login">Login</Link>
      )}
    </nav>
  )
}

Auth Patterns Summary

PatternWhen to useImplementation
Per-route guard One or two protected routes beforeLoad on each route
Layout-level guard Many protected routes _authenticated.tsx pathless layout
Nested role guard Role-based sections _authenticated/_admin.tsx
Redirect after login Always Save URL in search params, navigate after login
Python parallel: Layout-level guards are like applying @login_required to a Flask Blueprint. Every route in the blueprint is automatically protected. TanStack Router's pathless layouts achieve the same pattern, with the added benefit of shared UI (headers, sidebars) for the protected section.
