09 — Routes & Navigation

Defining routes, type-safe links, URL params, search params with Zod, nested layouts, and programmatic navigation.

Route Definitions

Every route file exports a Route object created with createFileRoute. The string you pass matches the file's position in the src/routes/ tree. The Vite plugin auto-generates this string, so you get a compile error if the path doesn't match.

// src/routes/users/index.tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/users/')({
  component: UsersPage,
})

function UsersPage() {
  return (
    <div>
      <h1>All Users</h1>
      {/* user list here */}
    </div>
  )
}
Route options (what you can pass to createFileRoute)
OptionPurpose
componentThe React component to render
loaderFetch data before rendering (covered in data-loading.html)
validateSearchZod schema for search/query params
beforeLoadGuard: runs before loader (auth checks)
errorComponentRendered when loader/component throws
pendingComponentRendered while loader is pending
notFoundComponentRendered for 404s within this route

The <Link> Component

Never use <a href="..."> for internal navigation. That would trigger a full page reload, losing all React state. Use <Link> instead — it intercepts the click, updates the URL via the History API, and lets React handle the transition.

Wrong: HTML anchor
<!-- Full page reload! -->
<!-- All React state lost -->
<a href="/users/42">View User</a>
Correct: Link component
{/* SPA navigation */}
{/* Type-safe: TS errors if path invalid */}
<Link
  to="/users/$userId"
  params={{ userId: '42' }}
>
  View User
</Link>

Link examples

import { Link } from '@tanstack/react-router'

// Static route — simple
<Link to="/">Home</Link>
<Link to="/about">About</Link>

// Dynamic route — must provide params
<Link
  to="/users/$userId"
  params={{ userId: '42' }}
>
  View Profile
</Link>

// With search params (query string)
<Link
  to="/users"
  search={{ page: 2, sort: 'name' }}
>
  Page 2
</Link>
{/* Renders as: /users?page=2&sort=name */}

// With hash
<Link to="/docs" hash="installation">Jump to Install</Link>
Type safety in action: If you write <Link to="/users/$userId"> without the params prop, TypeScript will error. If you write params={{ usrId: '42' }} (typo), TypeScript will error. The router knows every valid route and its expected params.

URL Params (Dynamic Segments)

Dynamic segments are the $paramName parts in your route path. They capture a variable portion of the URL.

Flask
@app.route("/users/<user_id>")
def user(user_id: str):
    # user_id comes from the URL
    return get_user(user_id)
TanStack Router
// src/routes/users/$userId.tsx
// File name IS the param

export const Route = createFileRoute(
  '/users/$userId'
)({
  component: UserDetail,
})

Accessing params with useParams()

// Inside the route component:
import { createFileRoute, useParams } from '@tanstack/react-router'

export const Route = createFileRoute('/users/$userId')({
  component: UserDetail,
})

function UserDetail() {
  // Fully typed! TypeScript knows userId is a string.
  const { userId } = Route.useParams()

  return <h1>User: {userId}</h1>
}

// URL: /users/42
// userId === "42"  (always a string, just like Flask)
Params are always strings. Just like Flask/FastAPI path params, URL segments are strings. If you need a number, parse it: Number(userId) or validate in the loader.

Search Params with Zod

Search params are the ?key=value portion of the URL. In most frameworks, these are untyped strings. TanStack Router lets you validate and type them with Zod — your schema defines the shape, defaults, and types.

FastAPI (Pydantic)
from fastapi import Query

@app.get("/users")
def list_users(
    page: int = Query(1),
    sort: str = Query("name"),
):
    # Validated & typed!
    ...
TanStack Router (Zod)
import { z } from 'zod'

const searchSchema = z.object({
  page: z.number().default(1),
  sort: z.string().default('name'),
})

// Validated & typed!

Full search params example

// src/routes/users/index.tsx
import { createFileRoute } from '@tanstack/react-router'
import { z } from 'zod'

const usersSearchSchema = z.object({
  page: z.number().catch(1),           // default 1, ignore bad values
  sort: z.enum(['name', 'date']).catch('name'),
  filter: z.string().optional(),       // not required
})

export const Route = createFileRoute('/users/')({
  validateSearch: usersSearchSchema,     // <-- Zod schema here
  component: UsersPage,
})

function UsersPage() {
  // Fully typed! { page: number, sort: 'name' | 'date', filter?: string }
  const { page, sort, filter } = Route.useSearch()

  return (
    <div>
      <p>Page: {page}, Sort: {sort}</p>
      {filter && <p>Filtering by: {filter}</p>}

      {/* Links update search params */}
      <Link
        to="/users"
        search={(prev) => ({ ...prev, page: prev.page + 1 })}
      >
        Next Page
      </Link>
    </div>
  )
}
Why .catch() instead of .default()? In search params, .catch(defaultValue) silently uses the default if the URL value is invalid (e.g., ?page=abc). .default() only applies when the key is missing. Use .catch() for search params — it's more resilient to user-edited URLs.

Nested Layouts

Layouts let you share UI (navigation, sidebars, headers) across multiple routes. Child routes render inside the parent's <Outlet />.

Jinja2 template inheritance
{# base.html #}
<nav>...</nav>
{% block content %}{% endblock %}
<footer>...</footer>

{# users.html #}
{% extends "base.html" %}
{% block content %}
  <h1>Users</h1>
{% endblock %}
TanStack Router layouts
// __root.tsx (parent)
<nav>...</nav>
<Outlet />  {/* child here */}
<footer>...</footer>

// routes/users.tsx renders
// inside that Outlet

Layout route example

// src/routes/dashboard.tsx — This is BOTH a layout AND a route
import { createFileRoute, Outlet, Link } from '@tanstack/react-router'

export const Route = createFileRoute('/dashboard')({
  component: DashboardLayout,
})

function DashboardLayout() {
  return (
    <div style={{ display: 'flex' }}>
      <aside>
        <Link to="/dashboard/analytics">Analytics</Link>
        <Link to="/dashboard/settings">Settings</Link>
      </aside>
      <main>
        <Outlet />  {/* analytics or settings renders here */}
      </main>
    </div>
  )
}

// src/routes/dashboard/analytics.tsx
export const Route = createFileRoute('/dashboard/analytics')({
  component: () => <h2>Analytics</h2>,
})

// src/routes/dashboard/settings.tsx
export const Route = createFileRoute('/dashboard/settings')({
  component: () => <h2>Settings</h2>,
})

Pathless layouts (_layout prefix)

Sometimes you want a layout that groups routes without adding a URL segment. Prefix the file/folder with _:

// File: src/routes/_authenticated.tsx
// This does NOT add "/authenticated" to the URL
// It just wraps child routes in a shared layout

export const Route = createFileRoute('/_authenticated')({
  component: () => (
    <div>
      <AuthHeader />
      <Outlet />
    </div>
  ),
})

// src/routes/_authenticated/dashboard.tsx  →  URL: /dashboard
// src/routes/_authenticated/settings.tsx   →  URL: /settings

Programmatic Navigation

Sometimes you need to navigate in response to an event (form submit, button click, after an API call) rather than a link click. Use useNavigate().

Flask (redirect)
from flask import redirect

@app.post("/login")
def login():
    # After auth...
    return redirect("/dashboard")
TanStack Router
const navigate = useNavigate()

const handleLogin = async () => {
  await login(credentials)
  navigate({ to: '/dashboard' })
}

useNavigate() examples

import { useNavigate } from '@tanstack/react-router'

function CreateUserForm() {
  const navigate = useNavigate()

  const onSubmit = async (data: FormData) => {
    const user = await createUser(data)

    // Navigate to the new user's page
    navigate({
      to: '/users/$userId',
      params: { userId: user.id },
    })
  }

  // Navigate with search params
  const goToPage = (page: number) => {
    navigate({
      to: '/users',
      search: { page },
    })
  }

  // Replace history (back button skips this page)
  navigate({
    to: '/dashboard',
    replace: true,  // like Flask's redirect with 302 vs 301
  })
}

Active Links

Style the currently active navigation link so users know where they are. <Link> supports activeProps and activeOptions.

// Active link with inline styles
<Link
  to="/users"
  activeProps={{
    style: { fontWeight: 'bold', color: '#e94560' },
  }}
>
  Users
</Link>

// Active link with CSS class
<Link
  to="/users"
  activeProps={{ className: 'nav-active' }}
  inactiveProps={{ className: 'nav-link' }}
>
  Users
</Link>

// Exact matching (default: true for index routes)
<Link
  to="/users"
  activeOptions={{
    exact: true,            // only /users, not /users/42
    includeSearch: false,   // ignore search params for matching
  }}
  activeProps={{ className: 'active' }}
>
  Users
</Link>

Building a nav bar

const NavBar = () => {
  const links = [
    { to: '/', label: 'Home' },
    { to: '/users', label: 'Users' },
    { to: '/about', label: 'About' },
  ] as const

  return (
    <nav>
      {links.map((link) => (
        <Link
          key={link.to}
          to={link.to}
          activeProps={{ className: 'active' }}
          activeOptions={{ exact: link.to === '/' }}
        >
          {link.label}
        </Link>
      ))}
    </nav>
  )
}

Full Example: Blog App Routes

Complete file structure
src/routes/
  __root.tsx ← Nav bar + <Outlet />
  index.tsx ← / Home page
  posts/
    index.tsx ← /posts List with search params
    $postId.tsx ← /posts/:postId Single post 
// src/routes/posts/index.tsx — List with pagination
import { createFileRoute, Link } from '@tanstack/react-router'
import { z } from 'zod'

const searchSchema = z.object({
  page: z.number().catch(1),
  tag: z.string().optional(),
})

export const Route = createFileRoute('/posts/')({
  validateSearch: searchSchema,
  component: PostList,
})

function PostList() {
  const { page, tag } = Route.useSearch()

  return (
    <div>
      <h1>Posts {tag && <span>tagged: {tag}</span>}</h1>

      {/* Post list would go here */}

      <Link
        to="/posts"
        search={(prev) => ({ ...prev, page: prev.page + 1 })}
      >
        Next &rarr;
      </Link>
    </div>
  )
}

// src/routes/posts/$postId.tsx — Single post
export const Route = createFileRoute('/posts/$postId')({
  component: PostDetail,
})

function PostDetail() {
  const { postId } = Route.useParams()
  const navigate = useNavigate()

  return (
    <article>
      <h1>Post #{postId}</h1>
      <button onClick={() => navigate({ to: '/posts' })}>
        &larr; Back to list
      </button>
    </article>
  )
}
Key pattern: Use Route.useParams() and Route.useSearch() (called on the Route object) for the best type inference. The generic useParams() from the router package works too, but requires you to specify the route type manually.
