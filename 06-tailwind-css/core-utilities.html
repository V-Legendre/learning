Pattern: p = padding, m = margin. x = horizontal, y = vertical. t/r/b/l = top/right/bottom/left. Same modifiers apply to both padding and margin.
Negative margins: Prefix with a dash: -mt-4 = margin-top: -1rem. Useful for overlapping elements.

Responsive Design

Tailwind is mobile-first. Unprefixed classes apply to all screen sizes. Prefixed classes apply at that breakpoint and above.

PrefixMin-widthTypical device
(none)0pxAll / mobile
sm:640pxLarge phones / small tablets
md:768pxTablets
lg:1024pxLaptops
xl:1280pxDesktops
2xl:1536pxLarge screens
 
{/* Mobile: 1 column. Tablet+: 2 columns. Desktop+: 3 columns */}
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <div>Card</div>
  <div>Card</div>
  <div>Card</div>
</div>

{/* Mobile: stack. Desktop: side by side */}
<div className="flex flex-col lg:flex-row gap-4">
  <aside className="lg:w-64">Sidebar</aside>
  <main className="flex-1">Content</main>
</div>

{/* Hide on mobile, show on desktop */}
<div className="hidden lg:block">Desktop only</div>

{/* Change text size at breakpoints */}
<h1 className="text-2xl md:text-4xl lg:text-5xl">Responsive Heading</h1>
Mobile-first means: Write your base styles for mobile, then add md:/lg: overrides. Don't start with desktop and add sm: overrides downward.

Dark Mode

Use the dark: prefix to apply styles only when dark mode is active. By default in v4, Tailwind uses the prefers-color-scheme media query. You can switch to class-based strategy if you want manual control.

{/* Adapts to system dark mode */}
<div className="bg-white dark:bg-gray-900 text-gray-900 dark:text-gray-100">
  <p className="text-gray-600 dark:text-gray-400">
    This text adapts to dark mode automatically.
  </p>
</div>

For class-based dark mode control, add this to your CSS:

/* src/index.css */
@import "tailwindcss";

@custom-variant dark (&:where(.dark, .dark *));

Then toggle by adding/removing the dark class on <html>:

<html class="dark"> <!-- dark mode on -->
<html class="">     <!-- light mode -->
