07 — TypeScript Patterns for React

Advanced TS techniques that make React components type-safe, self-documenting, and impossible to misuse. These patterns show why TypeScript + React is greater than the sum of its parts.

Discriminated Union Props

One of the most powerful patterns in TypeScript React. A component's props change shape based on a "tag" field. TypeScript narrows the type automatically, so you can't pass invalid combinations.

Python analogy: Tagged unions / Literal types
from typing import Literal, Union
from dataclasses import dataclass

@dataclass
class TextInput:
    variant: Literal["text"]
    value: str
    placeholder: str

@dataclass
class NumberInput:
    variant: Literal["number"]
    value: int
    min: int
    max: int

InputConfig = Union[TextInput, NumberInput]
TypeScript: Discriminated union
type InputProps =
  | {
      variant: "text"
      value: string
      placeholder?: string
    }
  | {
      variant: "number"
      value: number
      min?: number
      max?: number
    }
  | {
      variant: "select"
      value: string
      options: string[]
    }
The component narrows based on variant
const Input = (props: InputProps) => {
  switch (props.variant) {
    case "text":
      // TypeScript knows: props.placeholder exists, props.min does NOT
      return <input type="text" value={props.value} placeholder={props.placeholder} />
    case "number":
      // TypeScript knows: props.min, props.max exist
      return <input type="number" value={props.value} min={props.min} max={props.max} />
    case "select":
      // TypeScript knows: props.options exists
      return (
        <select value={props.value}>
          {props.options.map(opt => (
            <option key={opt} value={opt}>{opt}</option>
          ))}
        </select>
      )
  }
}

Usage — TypeScript catches invalid combinations

Valid: Props match variant
<Input variant="text" value="hello" />
<Input variant="number" value={42} min={0} />
<Input variant="select" value="a"
  options={["a", "b"]} />
Type error: Invalid combinations
// Error: 'min' doesn't exist on text
<Input variant="text" value="hi" min={0} />

// Error: value must be number for "number"
<Input variant="number" value="hi" />

// Error: 'options' required for "select"
<Input variant="select" value="a" />
This is the killer pattern. Use discriminated unions whenever a component has modes, variants, or types that change which props are valid. It makes impossible states impossible.

Event Handler Types

React wraps native DOM events in SyntheticEvent wrappers. The type includes which element triggered the event. You'll use these constantly for forms, clicks, and keyboard handling.

EventTypeCommon use
Click React.MouseEvent<HTMLButtonElement> Button clicks, link clicks
Input change React.ChangeEvent<HTMLInputElement> Text input, checkbox, radio
Select change React.ChangeEvent<HTMLSelectElement> Dropdown selection
Form submit React.FormEvent<HTMLFormElement> Form submission
Key press React.KeyboardEvent<HTMLInputElement> Keyboard shortcuts, enter key
Focus/Blur React.FocusEvent<HTMLInputElement> Validation on blur
Common event handler patterns
// Click handler
const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => {
  e.preventDefault()
  console.log("clicked", e.currentTarget.name)
}

// Input change — e.target.value is typed as string
const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  setQuery(e.target.value)
}

// Form submission
const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
  e.preventDefault()  // prevent page reload
  const formData = new FormData(e.currentTarget)
  // process form...
}

// Keyboard handler — check which key
const handleKeyDown = (e: React.KeyboardEvent<HTMLInputElement>) => {
  if (e.key === "Enter") submitSearch()
  if (e.key === "Escape") clearInput()
}
Shortcut: Instead of typing the event parameter, you can type the handler function itself: const handleChange: React.ChangeEventHandler<HTMLInputElement> = (e) => { ... }. TypeScript infers e automatically.

Generic Components

Components that work with any data type while maintaining type safety. Like Python's Generic[T] from typing, but for JSX.

Python generic
from typing import Generic, TypeVar, Callable

T = TypeVar("T")

class DataList(Generic[T]):
    def __init__(
        self,
        items: list[T],
        render_item: Callable[[T], str],
    ):
        self.items = items
        self.render = render_item
React generic component
type ListProps<T> = {
  items: T[]
  renderItem: (item: T) => React.ReactNode
  keyExtractor: (item: T) => string
}

const List = <T,>({
  items,
  renderItem,
  keyExtractor,
}: ListProps<T>) => (
  <ul>
    {items.map(item => (
      <li key={keyExtractor(item)}>
        {renderItem(item)}
      </li>
    ))}
  </ul>
)

Usage — TypeScript infers T from the items

type User = { id: string; name: string; email: string }

// T is inferred as User from the `items` prop
<List
  items={users}
  keyExtractor={user => user.id}
  renderItem={user => (
    // `user` is fully typed as User here
    <span>{user.name} ({user.email})</span>
  )}
/>
Syntax note: The trailing comma in <T,> is required in .tsx files. Without it, the parser confuses the generic <T> with a JSX tag. In .ts files, <T> works fine.

React.ComponentProps

Extract the props type from any component without importing its props type directly. Essential when wrapping third-party components or building higher-order components.

// Extract props from a custom component
type ButtonProps = React.ComponentProps<typeof Button>

// Extract props from an HTML element
type InputProps = React.ComponentProps<"input">
type DivProps = React.ComponentProps<"div">

// Practical use: wrap a component and add props
type EnhancedButtonProps = React.ComponentProps<typeof Button> & {
  isLoading?: boolean
}

const EnhancedButton = ({ isLoading, children, ...rest }: EnhancedButtonProps) => (
  <Button {...rest} disabled={isLoading || rest.disabled}>
    {isLoading ? <Spinner /> : children}
  </Button>
)

ReactNode vs ReactElement vs JSX.Element

Three similar types that often cause confusion. Here's when to use each.

TypeIncludesUse when
React.ReactNode Everything: elements, strings, numbers, null, undefined, booleans, arrays Accepting children, any renderable prop
React.ReactElement Only JSX elements (not strings, numbers, null) When you specifically need a JSX element (e.g., cloneElement)
JSX.Element Same as ReactElement but less explicit Rarely. Prefer ReactElement 
// ReactNode: accepts anything renderable (most common)
type CardProps = { children: React.ReactNode }

// ReactElement: must be actual JSX (not a string)
type TabProps = { icon: React.ReactElement }
// Usage: <Tab icon={<HomeIcon />} />  OK
// Usage: <Tab icon="home" />  ERROR
Default to React.ReactNode unless you have a specific reason to require a JSX element. It's the most flexible and the standard for children.

Extending HTML Elements

When building a custom Button, Input, etc., you want it to accept all the native HTML attributes plus your custom props. Extend the HTML attribute types.

Custom button that extends native <button>
type ButtonProps = React.ButtonHTMLAttributes<HTMLButtonElement> & {
  variant?: "primary" | "secondary" | "danger"
  size?: "sm" | "md" | "lg"
  isLoading?: boolean
}

const Button = ({
  variant = "primary",
  size = "md",
  isLoading,
  className,
  children,
  disabled,
  ...rest  // all remaining native button props
}: ButtonProps) => (
  <button
    className={`btn btn--${variant} btn--${size} ${className ?? ""}`}
    disabled={disabled || isLoading}
    {...rest}  {/* spreads onClick, type, aria-*, etc. */}
  >
    {isLoading ? <Spinner /> : children}
  </button>
)

Now it accepts everything a native button does

<Button variant="danger" onClick={handleDelete}>Delete</Button>
<Button type="submit" form="myForm">Save</Button>
<Button aria-label="Close" disabled>X</Button>
Common HTML attribute types
ElementAttribute type
<button>React.ButtonHTMLAttributes<HTMLButtonElement>
<input>React.InputHTMLAttributes<HTMLInputElement>
<a>React.AnchorHTMLAttributes<HTMLAnchorElement>
<div>React.HTMLAttributes<HTMLDivElement>
Any elementReact.ComponentProps<"elementName">

as const Assertions

Make values read-only with literal types instead of wide types. Crucial for config objects and dropdown options.

Without as const
const SIZES = ["sm", "md", "lg"]
// Type: string[]
// TypeScript only knows it's an array of strings.
// Can't use as a union type.

const CONFIG = { maxRetries: 3 }
// Type: { maxRetries: number }
// Could be mutated. Type is wide.
With as const
const SIZES = ["sm", "md", "lg"] as const
// Type: readonly ["sm", "md", "lg"]
// Now we can derive a union type:
type Size = (typeof SIZES)[number]
// Type: "sm" | "md" | "lg"

const CONFIG = { maxRetries: 3 } as const
// Type: { readonly maxRetries: 3 }
// Immutable. Literal type 3, not number.
Practical use: Dropdown from as const array
const ROLES = ["admin", "editor", "viewer"] as const
type Role = (typeof ROLES)[number]  // "admin" | "editor" | "viewer"

type RoleSelectProps = {
  value: Role
  onChange: (role: Role) => void
}

const RoleSelect = ({ value, onChange }: RoleSelectProps) => (
  <select
    value={value}
    onChange={e => onChange(e.target.value as Role)}
  >
    {ROLES.map(role => (
      <option key={role} value={role}>
        {role.charAt(0).toUpperCase() + role.slice(1)}
      </option>
    ))}
  </select>
)

// Single source of truth: the ROLES array defines both
// the type AND the runtime values. No duplication.

Pattern Summary

PatternWhen to UsePython Equivalent
Discriminated unions Component has modes/variants with different valid props Union[TypeA, TypeB] with a Literal discriminant
Event types Any event handler (click, change, submit, keydown) No direct analog — Python web frameworks abstract events away
Generic components Data-agnostic components (List, Table, Select) class MyClass(Generic[T])
ComponentProps Wrapping existing components, adding props get_type_hints(some_function)
Extending HTML Custom components that wrap native elements No equivalent — Python doesn't have HTML attributes
as const Config objects, dropdown options, route maps tuple + Literal types, or Enum
