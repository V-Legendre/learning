08 — Custom Hooks

Extracting reusable stateful logic from components. The React equivalent of Python utility functions, but with superpowers.

What Is a Custom Hook?

A custom hook is a regular function that starts with use and calls other hooks. It extracts stateful logic out of a component into a reusable function. The component provides UI; the hook provides behavior.

Python analogy: Reusable behavior
# Decorator adds behavior to any function
def retry(max_attempts=3):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for i in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except Exception:
                    if i == max_attempts - 1:
                        raise
        return wrapper
    return decorator

# Mixin adds behavior to any class
class TimestampMixin:
    created_at = Column(DateTime)
    updated_at = Column(DateTime)
React: Custom hook
// Custom hook adds behavior to any component
const useToggle = (initial = false) => {
  const [value, setValue] = useState(initial)
  const toggle = () => setValue(v => !v)
  const setTrue = () => setValue(true)
  const setFalse = () => setValue(false)
  return { value, toggle, setTrue, setFalse }
}

// Use in any component
const Modal = () => {
  const { value: isOpen, toggle } = useToggle()
  // ...
}

const Sidebar = () => {
  const { value: expanded, toggle } = useToggle(true)
  // ...
}

Rules of Custom Hooks

RuleWhy
Name must start with use React's linter enforces hook rules based on this prefix. Without it, hooks inside won't be checked.
Can call any hook (useState, useEffect, other custom hooks) That's the whole point — encapsulate hook calls
Follows all hook rules (no conditionals, no loops) Hooks must be called in the same order every render
Each call gets its own state Two components using the same hook get independent state copies. Hooks share logic, not state.
Custom hooks share logic, NOT state. If component A and component B both call useToggle(), they each get their own independent useState. To share state, you need Context or a state manager.

Example: useLocalStorage

Persist state to localStorage. Reads the initial value from storage, and syncs every update back. Fully generic.

useLocalStorage implementation
import { useState, useEffect } from "react"

export const useLocalStorage = <T,>(
  key: string,
  initialValue: T,
): [T, (value: T | ((prev: T) => T)) => void] => {
  // Read from localStorage on first render
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key)
      return item ? JSON.parse(item) : initialValue
    } catch {
      return initialValue
    }
  })

  // Sync to localStorage whenever value changes
  useEffect(() => {
    window.localStorage.setItem(key, JSON.stringify(storedValue))
  }, [key, storedValue])

  return [storedValue, setStoredValue]
}

// Usage — same API as useState, but persisted!
const [theme, setTheme] = useLocalStorage<"light" | "dark">("theme", "light")
const [favorites, setFavorites] = useLocalStorage<string[]>("favorites", [])

Example: useDebounce

Debounce a rapidly changing value (e.g., search input). Only returns the latest value after the user stops typing for a specified delay.

useDebounce implementation
import { useState, useEffect } from "react"

export const useDebounce = <T,>(value: T, delayMs: number): T => {
  const [debouncedValue, setDebouncedValue] = useState(value)

  useEffect(() => {
    const timer = setTimeout(() => {
      setDebouncedValue(value)
    }, delayMs)

    // Cleanup: cancel previous timer if value changes again
    return () => clearTimeout(timer)
  }, [value, delayMs])

  return debouncedValue
}

// Usage: search input that doesn't fire on every keystroke
const SearchPage = () => {
  const [query, setQuery] = useState("")
  const debouncedQuery = useDebounce(query, 300) // 300ms delay

  // Only fetches when user stops typing for 300ms
  const { data } = useQuery({
    queryKey: ["search", debouncedQuery],
    queryFn: () => searchAPI(debouncedQuery),
    enabled: debouncedQuery.length > 0,
  })

  return <input value={query} onChange={e => setQuery(e.target.value)} />
}

Example: useMediaQuery

Reactive responsive design. Returns whether a CSS media query currently matches. Updates when the window is resized.

useMediaQuery implementation
import { useState, useEffect } from "react"

export const useMediaQuery = (query: string): boolean => {
  const [matches, setMatches] = useState(() =>
    window.matchMedia(query).matches
  )

  useEffect(() => {
    const mediaQuery = window.matchMedia(query)
    const handler = (e: MediaQueryListEvent) => setMatches(e.matches)

    mediaQuery.addEventListener("change", handler)
    return () => mediaQuery.removeEventListener("change", handler)
  }, [query])

  return matches
}

// Usage
const Dashboard = () => {
  const isMobile = useMediaQuery("(max-width: 768px)")
  const prefersDark = useMediaQuery("(prefers-color-scheme: dark)")

  return isMobile ? <MobileLayout /> : <DesktopLayout />
}

Example: useOnClickOutside

Detect clicks outside a referenced element. Essential for closing dropdowns, modals, and popovers when the user clicks away.

useOnClickOutside implementation
import { useEffect, useRef } from "react"

export const useOnClickOutside = (
  ref: React.RefObject<HTMLElement | null>,
  handler: () => void,
) => {
  useEffect(() => {
    const listener = (event: MouseEvent) => {
      // Do nothing if clicking ref's element or its descendants
      if (!ref.current || ref.current.contains(event.target as Node)) {
        return
      }
      handler()
    }

    document.addEventListener("mousedown", listener)
    return () => document.removeEventListener("mousedown", listener)
  }, [ref, handler])
}

// Usage: dropdown that closes on outside click
const Dropdown = () => {
  const [isOpen, setIsOpen] = useState(false)
  const dropdownRef = useRef<HTMLDivElement>(null)

  useOnClickOutside(dropdownRef, () => setIsOpen(false))

  return (
    <div ref={dropdownRef}>
      <button onClick={() => setIsOpen(!isOpen)}>Menu</button>
      {isOpen && (
        <ul className="dropdown-menu">
          <li>Option 1</li>
          <li>Option 2</li>
        </ul>
      )}
    </div>
  )
}

Example: useFetch (and Why TanStack Query Is Better)

Simplified useFetch hook
export const useFetch = <T,>(url: string) => {
  const [data, setData] = useState<T | null>(null)
  const [isLoading, setIsLoading] = useState(true)
  const [error, setError] = useState<string | null>(null)

  useEffect(() => {
    const controller = new AbortController()

    const fetchData = async () => {
      try {
        setIsLoading(true)
        const response = await fetch(url, { signal: controller.signal })
        if (!response.ok) throw new Error(`HTTP ${response.status}`)
        const json = await response.json()
        setData(json)
        setError(null)
      } catch (e) {
        if (e instanceof DOMException && e.name === "AbortError") return
        setError(e instanceof Error ? e.message : "Unknown error")
      } finally {
        setIsLoading(false)
      }
    }

    fetchData()
    return () => controller.abort()
  }, [url])

  return { data, isLoading, error }
}
Don't use useFetch in production. It's missing: caching, deduplication, background refetching, optimistic updates, infinite queries, retry logic, garbage collection, devtools. TanStack Query handles all of this. Build useFetch to learn hooks; use TanStack Query to ship products.

Composition: Hooks Calling Hooks

Custom hooks can call other custom hooks. Build small hooks, then compose them into more powerful ones.

Composing hooks together
// Base hooks
const useToggle = (initial = false) => {
  const [value, setValue] = useState(initial)
  const toggle = () => setValue(v => !v)
  return { value, toggle, setValue }
}

const useOnClickOutside = (ref: React.RefObject<HTMLElement | null>, handler: () => void) => {
  // ... (see above)
}

// Composed hook: combines toggle + click outside + escape key
const useDropdown = () => {
  const { value: isOpen, toggle, setValue: setIsOpen } = useToggle()
  const ref = useRef<HTMLDivElement>(null)

  useOnClickOutside(ref, () => setIsOpen(false))

  useEffect(() => {
    const handleEscape = (e: KeyboardEvent) => {
      if (e.key === "Escape") setIsOpen(false)
    }
    document.addEventListener("keydown", handleEscape)
    return () => document.removeEventListener("keydown", handleEscape)
  }, [setIsOpen])

  return { isOpen, toggle, ref }
}

// Usage is clean and declarative
const UserMenu = () => {
  const { isOpen, toggle, ref } = useDropdown()

  return (
    <div ref={ref}>
      <button onClick={toggle}>Menu</button>
      {isOpen && <DropdownMenu />}
    </div>
  )
}

Testing Custom Hooks

Use renderHook from @testing-library/react to test hooks in isolation, without building a dummy component.

import { renderHook, act } from "@testing-library/react"
import { useToggle } from "./useToggle"

test("useToggle starts with initial value", () => {
  const { result } = renderHook(() => useToggle(false))
  expect(result.current.value).toBe(false)
})

test("toggle flips the value", () => {
  const { result } = renderHook(() => useToggle(false))

  act(() => {
    result.current.toggle()
  })

  expect(result.current.value).toBe(true)
})

test("useDebounce returns debounced value", async () => {
  const { result, rerender } = renderHook(
    ({ value, delay }) => useDebounce(value, delay),
    { initialProps: { value: "hello", delay: 300 } },
  )

  expect(result.current).toBe("hello")

  // Change the value
  rerender({ value: "world", delay: 300 })

  // Still the old value (debouncing)
  expect(result.current).toBe("hello")

  // Wait for debounce
  await waitFor(() => {
    expect(result.current).toBe("world")
  })
})

Custom Hook Patterns Summary

HookPurposeKey hooks used inside
useToggle Boolean on/off state useState
useLocalStorage Persist state to browser storage useState, useEffect
useDebounce Delay rapidly changing values useState, useEffect
useMediaQuery Responsive design in JS useState, useEffect
useOnClickOutside Close dropdowns/modals useEffect, useRef
useDropdown Full dropdown behavior useToggle, useOnClickOutside, useEffect
When to extract a custom hook: (1) You copy-paste the same useState + useEffect combo in multiple components. (2) A component has complex hook logic that obscures the UI code. (3) You want to test stateful logic in isolation. The hook doesn't need to be reusable to be worth extracting — clarity alone justifies it.
