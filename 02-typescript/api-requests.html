API Requests in TypeScript + React

From raw fetch to TanStack Query + Zod — the modern data fetching stack

The Modern Stack (2025-2026)

TL;DR: Use TanStack Query for fetching/caching, Zod for runtime validation, and plain fetch (or ky) as the HTTP client. Don't use useEffect for data fetching. Don't use Redux for server state.
ConcernUseAvoid
Server state (API data)TanStack QueryuseEffect + useState, Redux
HTTP clientfetch (native) or kyaxios (unnecessary overhead)
Runtime validationZodTrusting as Type assertions
Client state (UI)useState or ZustandRedux for simple state

Step 1: Raw fetch in TypeScript

Let's start with the basics before adding libraries.

Python (requests)
import requests

response = requests.get("/api/users/1")
response.raise_for_status()  # throws on 4xx/5xx
user: dict = response.json()
TypeScript (fetch)
const response = await fetch("/api/users/1")

if (!response.ok) {          // must check manually!
  throw new Error(`HTTP ${response.status}`)
}

const user: User = await response.json()
Key difference from Python's requests
  • fetch does NOT throw on 4xx/5xx — a 404 is a "successful" HTTP response. You must check response.ok yourself.
  • response.json() returns Promise<any> — TypeScript has no idea what the API returns. The : User annotation is a hint, not a guarantee.
  • This is the exact problem Zod solves (see below).

A typed fetch wrapper

const fetchJson = async <T>(url: string, options?: RequestInit): Promise<T> => {
  const response = await fetch(url, options)

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${response.statusText}`)
  }

  return response.json() as Promise<T>
}

// Usage — T is inferred from the generic
const user = await fetchJson<User>("/api/users/1")
const posts = await fetchJson<Post[]>("/api/posts")

Step 2: Zod — Runtime Validation

Python (Pydantic)
from pydantic import BaseModel

class User(BaseModel):
    id: int
    name: str
    email: str

# Validates at runtime — throws on bad data
user = User(**response.json())
TypeScript (Zod)
import { z } from "zod"

const UserSchema = z.object({
  id: z.number(),
  name: z.string(),
  email: z.string().email(),
})

// Infer the TS type FROM the schema
type User = z.infer<typeof UserSchema>

// Validates at runtime — throws on bad data
const user = UserSchema.parse(apiResponse)
Zod is TypeScript's Pydantic. You define the schema once, and get both the runtime validator AND the TypeScript type from it. Single source of truth.

Safe parsing (no throw)

// .parse() throws on invalid data
// .safeParse() returns a discriminated union instead
const result = UserSchema.safeParse(apiResponse)

if (result.success) {
  console.log(result.data)         // User type, validated
} else {
  console.error(result.error.issues) // array of validation errors
}

Composing schemas (like Pydantic model inheritance)

// Remove fields
const CreateUserSchema = UserSchema.omit({ id: true })

// Make all fields optional
const PartialUserSchema = UserSchema.partial()

// Pick specific fields
const UserNameSchema = UserSchema.pick({ name: true, email: true })

// Unions, enums, literals
const ResponseSchema = z.discriminatedUnion("type", [
  z.object({ type: z.literal("success"), data: UserSchema }),
  z.object({ type: z.literal("error"), message: z.string() }),
])

Step 3: TanStack Query — Server State Management

This is the core library. It handles caching, deduplication, background refresh, retries, loading/error states — all the things you'd otherwise build by hand.

What it replaces

Without TanStack Query, every component that fetches data needs:

  • useState for data
  • useState for loading
  • useState for error
  • useEffect for the fetch call
  • Manual cleanup / abort controller
  • No caching, no deduplication
  • Race conditions on fast navigation
Python analogy

Think of TanStack Query as a smart caching layer with:

  • TTLs (staleTime)
  • Automatic invalidation
  • Retry logic
  • Background refresh
  • Garbage collection (gcTime)

Like Redis + Celery for your API calls, but built into React's component model.

Setup

// main.tsx — wrap your app once
import { QueryClient, QueryClientProvider } from "@tanstack/react-query"

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 2 * 60 * 1000,  // data is "fresh" for 2 minutes
      gcTime: 10 * 60 * 1000,    // unused cache lives 10 minutes
      retry: 3,                    // retry failed requests 3 times
    },
  },
})

const App = () => (
  <QueryClientProvider client={queryClient}>
    <YourApp />
  </QueryClientProvider>
)

useQuery — Reading data

import { useQuery } from "@tanstack/react-query"

const TodoList = () => {
  const {
    data,       // Todo[] | undefined
    isPending,  // true during first load (no cached data yet)
    isError,    // true if query failed
    error,      // the Error object
    isFetching, // true during ANY fetch (including background refetch)
  } = useQuery({
    queryKey: ["todos"],     // unique cache key (array)
    queryFn: fetchTodos,       // function that returns a Promise
  })

  if (isPending) return <div>Loading...</div>
  if (isError) return <div>Error: {error.message}</div>

  return (
    <ul>
      {data.map((todo) => (
        <li key={todo.id}>{todo.title}</li>
      ))}
    </ul>
  )
}

Query keys — how caching works

Query keys are arrays that identify cached data
useQuery({ queryKey: ["todos"] })                        // all todos
useQuery({ queryKey: ["todos", { status: "done" }] })   // filtered
useQuery({ queryKey: ["todos", todoId] })               // single todo
useQuery({ queryKey: ["users", userId, "posts"] })    // user's posts

Compared by deep equality. Invalidating ["todos"] invalidates ALL queries that start with "todos".

staleTime vs gcTime

SettingDefaultWhat it controls
staleTime 0 (immediately stale) How long data is "fresh." While fresh, no network request is made — cached data is returned instantly.
gcTime 5 min How long unused cache (no component subscribed) stays in memory before garbage collection.
The lifecycle
1. Component mounts → useQuery fires → data fetched and cached. Status: fresh
2. After staleTime passes → status becomes stale
3. Component re-mounts (or window regains focus) → stale data returned instantly, background refetch fires
4. No component subscribed + gcTime passes → cache entry deleted

useMutation — Writing data (POST/PUT/DELETE)

import { useMutation, useQueryClient } from "@tanstack/react-query"

const AddTodoButton = () => {
  const queryClient = useQueryClient()

  const mutation = useMutation({
    mutationFn: (newTodo: { title: string }) =>
      fetch("/api/todos", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(newTodo),
      }).then((res) => res.json()),

    onSuccess: () => {
      // Invalidate cache → triggers refetch of the todo list
      queryClient.invalidateQueries({ queryKey: ["todos"] })
    },
  })

  return (
    <button
      onClick={() => mutation.mutate({ title: "New task" })}
      disabled={mutation.isPending}
    >
      {mutation.isPending ? "Adding..." : "Add Todo"}
    </button>
  )
}

Step 4: Putting It All Together

The recommended project structure separates API logic into layers:

src/
  api/
    schemas.ts       # Zod schemas (single source of truth for types)
    client.ts        # configured fetch wrapper
    users.ts         # user API functions
    todos.ts         # todo API functions
  hooks/
    useUsers.ts      # TanStack Query hooks (what components import)
    useTodos.ts
  components/
    TodoList.tsx     # uses hooks — knows nothing about fetch/API

Layer 1: Schemas (Zod)

// api/schemas.ts
import { z } from "zod"

export const TodoSchema = z.object({
  id: z.number(),
  title: z.string(),
  completed: z.boolean(),
})

export const TodoListSchema = z.array(TodoSchema)
export type Todo = z.infer<typeof TodoSchema>

Layer 2: API functions (pure, no React)

// api/todos.ts
import { TodoListSchema, TodoSchema, type Todo } from "./schemas"

export const todosApi = {
  getAll: async (): Promise<Todo[]> => {
    const res = await fetch("/api/todos")
    if (!res.ok) throw new Error(`HTTP ${res.status}`)
    return TodoListSchema.parse(await res.json())  // validated!
  },

  create: async (title: string): Promise<Todo> => {
    const res = await fetch("/api/todos", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ title }),
    })
    if (!res.ok) throw new Error(`HTTP ${res.status}`)
    return TodoSchema.parse(await res.json())
  },
}

Layer 3: React hooks (TanStack Query)

// hooks/useTodos.ts
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query"
import { todosApi } from "../api/todos"

export const useTodos = () =>
  useQuery({
    queryKey: ["todos"],
    queryFn: todosApi.getAll,
  })

export const useCreateTodo = () => {
  const queryClient = useQueryClient()
  return useMutation({
    mutationFn: todosApi.create,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["todos"] })
    },
  })
}

Layer 4: Component (clean, no API logic)

// components/TodoList.tsx
import { useTodos, useCreateTodo } from "../hooks/useTodos"

export const TodoList = () => {
  const { data: todos, isPending, isError, error } = useTodos()
  const createTodo = useCreateTodo()

  if (isPending) return <div>Loading...</div>
  if (isError) return <div>Error: {error.message}</div>

  return (
    <div>
      <ul>
        {todos.map((todo) => (
          <li key={todo.id}>{todo.title}</li>
        ))}
      </ul>
      <button
        onClick={() => createTodo.mutate("New task")}
        disabled={createTodo.isPending}
      >
        {createTodo.isPending ? "Adding..." : "Add Todo"}
      </button>
    </div>
  )
}

What NOT to Do (Outdated Patterns)

DON'T — useEffect for fetching
// This was how everyone did it in 2019-2022
// It's now considered an anti-pattern

const [data, setData] = useState(null)
const [loading, setLoading] = useState(true)
const [error, setError] = useState(null)

useEffect(() => {
  fetch("/api/users")
    .then((res) => res.json())
    .then(setData)
    .catch(setError)
    .finally(() => setLoading(false))
}, [])

No caching, no dedup, race conditions, manual boilerplate in every component, memory leaks

DO — TanStack Query
// Handles everything automatically

const { data, isPending, isError, error } =
  useQuery({
    queryKey: ["users"],
    queryFn: fetchUsers,
  })

Caching, deduplication, background refresh, retries, loading/error states — all automatic

Other outdated patterns
  • Redux for server state — Redux is for client state (UI). TanStack Query handles API data.
  • axios as default choice — Native fetch is sufficient. axios adds bundle weight for features TanStack Query already handles (retries, cancellation).
  • as Type assertions on API data — Use Zod to validate. Type assertions are lies to the compiler.
