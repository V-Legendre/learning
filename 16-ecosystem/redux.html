16 — Redux

The OG state management library: what it is, how Redux Toolkit works, and why we chose Zustand instead.

What Is Redux?

Redux is a predictable state container for JavaScript apps. It was the dominant React state library for years (2015-2020) and is still used in many production codebases. Understanding it helps you read existing code and makes you appreciate simpler alternatives.

Core concepts

1
Store
Single object holds ALL app state
→
2
Action
Plain object describing what happened
→
3
Dispatch
Send action to the store
→
4
Reducer
Pure function: state + action = new state
→
5
Re-render
Components subscribed to state update
Redux principles
  • Single source of truth — one store, one state tree
  • State is read-only — the only way to change it is to dispatch an action
  • Changes via pure functions — reducers take (state, action) and return new state, no side effects
Python analogy: Think of Redux like an event-sourced system. Actions are events (like database audit logs). Reducers are event handlers. The store is the current projected state. You can replay all actions to reconstruct any point in time.

Classic Redux (the verbose way)

This is what Redux looked like before Redux Toolkit. You'll still see this in older codebases.

// Action types (string constants)
const INCREMENT = 'counter/increment'
const DECREMENT = 'counter/decrement'

// Action creators (functions that return action objects)
const increment = (amount: number) => ({
  type: INCREMENT,
  payload: amount,
})

// Reducer (pure function, switch statement)
const counterReducer = (
  state = { value: 0 },
  action: { type: string; payload?: number }
) => {
  switch (action.type) {
    case INCREMENT:
      return { ...state, value: state.value + (action.payload ?? 1) }
    case DECREMENT:
      return { ...state, value: state.value - 1 }
    default:
      return state
  }
}

// Usage in component
const value = useSelector((state) => state.counter.value)
const dispatch = useDispatch()
dispatch(increment(5))

For this trivial counter, that's about 30 lines. This "boilerplate problem" is why Redux got a bad reputation.

Redux Toolkit (RTK) — Modern Redux

Redux Toolkit is the official recommended way to write Redux. It dramatically reduces boilerplate.

// features/counter/counterSlice.ts
import { createSlice } from '@reduxjs/toolkit'
import type { PayloadAction } from '@reduxjs/toolkit'

interface CounterState {
  value: number
}

const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 } as CounterState,
  reducers: {
    // RTK uses Immer internally — you can "mutate" state directly
    increment(state, action: PayloadAction<number>) {
      state.value += action.payload
    },
    decrement(state) {
      state.value -= 1
    },
  },
})

// Auto-generated action creators and action types
export const { increment, decrement } = counterSlice.actions
export default counterSlice.reducer
// store.ts
import { configureStore } from '@reduxjs/toolkit'
import counterReducer from './features/counter/counterSlice'

export const store = configureStore({
  reducer: {
    counter: counterReducer,
  },
})

// Infer types from the store itself
export type RootState = ReturnType<typeof store.getState>
export type AppDispatch = typeof store.dispatch
Immer under the hood: Notice the state.value += action.payload? In classic Redux, you'd need to return a new object ({ ...state, value: state.value + action.payload }). RTK uses Immer to let you write "mutating" code that produces immutable updates.

RTK Query

RTK Query is Redux's answer to TanStack Query — data fetching with caching, built into the Redux store.

// features/api/apiSlice.ts
import { createApi, fetchBaseQuery } from '@reduxjs/toolkit/query/react'
import type { User } from '../types'

export const apiSlice = createApi({
  reducerPath: 'api',
  baseQuery: fetchBaseQuery({ baseUrl: '/api' }),
  endpoints: (builder) => ({
    getUsers: builder.query<User[], void>({
      query: () => '/users',
    }),
    getUserById: builder.query<User, string>({
      query: (id) => `/users/${id}`,
    }),
  }),
})

// Auto-generated hooks
export const { useGetUsersQuery, useGetUserByIdQuery } = apiSlice

When Redux Makes Sense

Redux is worth it for...
  • Very large teams — Redux enforces patterns. Everyone follows the same action/reducer/dispatch structure.
  • Complex shared state — many parts of the app reading and writing the same data with many different actions
  • Existing Redux codebase — if the app is already on Redux, migrating is often not worth it
  • Time-travel debugging — Redux DevTools can replay state changes step by step
  • Middleware ecosystem — saga, thunk, logging, analytics — mature middleware library
Skip Redux when...
  • Small to medium apps — the ceremony isn't worth it for most projects
  • Mostly server state — if 80% of your state is API data, TanStack Query handles it better
  • Small team — the enforced patterns are overhead without many developers
  • Starting a new project — Zustand + TanStack Query covers 95% of use cases with 90% less code

Why We Picked Zustand Over Redux

For our learning path, Zustand is the better choice. Here's the same counter implemented in both:

Redux Toolkit
// counterSlice.ts
import { createSlice } from '@reduxjs/toolkit'

const slice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment(state) {
      state.value += 1
    },
  },
})

export const { increment } = slice.actions
export default slice.reducer

// store.ts (separate file)
import { configureStore } from '...'

// Provider required in App.tsx
// useSelector + useDispatch in components
Zustand
// counterStore.ts
import { create } from 'zustand'

export const useCounter = create(() => ({
  value: 0,
  increment: () =>
    set((s) => ({
      value: s.value + 1
    })),
}))

// That's it. No store config.
// No Provider.
// Use directly in components:
// const value = useCounter(s => s.value)
// const increment = useCounter(s => s.increment)

Bottom line: Redux is not bad. Modern Redux (RTK) is actually quite good. But for most new projects, Zustand + TanStack Query gives you the same capabilities with dramatically less code. Learn Redux concepts (they're everywhere), but reach for Zustand in practice.
