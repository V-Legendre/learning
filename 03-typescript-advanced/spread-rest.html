03 — Spread & Rest Operators

The ... operator — spreads things apart or gathers them together. Core to immutable state updates in React.

Two Sides of the Same Coin

Spread vs Rest — same syntax, different context

The ... operator does two opposite things depending on where you use it:

  • Spread = "unpack/expand" an iterable or object into individual elements
  • Rest = "gather/collect" multiple elements into a single array or object
Spread (expanding)
const arr = [1, 2, 3]
const copy = [...arr]
// Unpacks [1, 2, 3] into 1, 2, 3
// then wraps in new array

const obj = { a: 1 }
const copy = { ...obj }
// Unpacks {a: 1} into a: 1
// then wraps in new object
Rest (collecting)
const [first, ...rest] = [1, 2, 3]
// rest = [2, 3]
// Gathers remaining into array

const { a, ...rest } = { a: 1, b: 2, c: 3 }
// rest = { b: 2, c: 3 }
// Gathers remaining into object

Spread Objects

Shallow Copy

TypeScript
const original = { x: 1, y: 2 }
const copy = { ...original }
// copy = { x: 1, y: 2 }
// Different reference — mutating copy
// doesn't affect original
Python equivalent
original = {"x": 1, "y": 2}
copy = {**original}
# copy = {"x": 1, "y": 2}
# Same: ** unpacks dict
# Also: dict(original) or original.copy()

Override Properties (Immutable Update)

This is the single most important pattern for React state updates. You create a new object with some properties changed.

const user = { name: "Alice", age: 30, role: "user" }

// Create new object with age changed
const updated = { ...user, age: 31 }
// { name: "Alice", age: 31, role: "user" }

// Add a new property
const withEmail = { ...user, email: "alice@example.com" }
// { name: "Alice", age: 30, role: "user", email: "alice@example.com" }

// Override multiple properties
const promoted = { ...user, role: "admin", age: 31 }
// { name: "Alice", age: 31, role: "admin" }
Order matters! Later properties override earlier ones. { ...user, age: 31 } works because age: 31 comes AFTER ...user. If reversed: { age: 31, ...user }, the original age: 30 would win.

Merging Objects

TypeScript
const defaults = {
  theme: "dark",
  lang: "en",
  debug: false
}
const userPrefs = {
  theme: "light",
  fontSize: 16
}

const config = { ...defaults, ...userPrefs }
// { theme: "light", lang: "en",
//   debug: false, fontSize: 16 }
// userPrefs overrides defaults
Python equivalent
defaults = {
    "theme": "dark",
    "lang": "en",
    "debug": False
}
user_prefs = {
    "theme": "light",
    "font_size": 16
}

config = {**defaults, **user_prefs}
# Same semantics — later wins
# Python 3.9+: defaults | user_prefs

Spread Arrays

// Concatenate arrays
const a = [1, 2]
const b = [3, 4]
const merged = [...a, ...b]        // [1, 2, 3, 4]

// Prepend / append
const items = ["b", "c"]
const withFirst = ["a", ...items]   // ["a", "b", "c"]
const withLast = [...items, "d"]    // ["b", "c", "d"]

// Insert in the middle
const nums = [1, 2, 5, 6]
const inserted = [...nums.slice(0, 2), 3, 4, ...nums.slice(2)]
// [1, 2, 3, 4, 5, 6]

// Shallow copy
const copy = [...items]             // new array, same elements
TypeScript spread
const merged = [...a, ...b]
const withNew = [...arr, item]
Python equivalent
merged = [*a, *b]
with_new = [*arr, item]
# Also: a + b, arr + [item]

Rest Parameters in Functions

TypeScript
// Gather all args into an array
const sum = (...nums: number[]): number => {
  return nums.reduce((a, b) => a + b, 0)
}

sum(1, 2, 3)     // 6
sum(10, 20)       // 30

// Mixed: fixed params + rest
const log = (
  level: string,
  ...messages: string[]
) => {
  console.log(`[${level}]`, ...messages)
}

log("ERROR", "Failed", "to connect")
Python equivalent
# *args gathers positional args
def sum_nums(*nums: int) -> int:
    return sum(nums)

sum_nums(1, 2, 3)    # 6
sum_nums(10, 20)      # 30

# Mixed: fixed + *args
def log(
    level: str,
    *messages: str
):
    print(f"[{level}]", *messages)

log("ERROR", "Failed", "to connect")
Note: JS/TS has ...args (three dots), Python has *args (one star). JS doesn't have **kwargs directly, but rest with object destructuring achieves the same: const { known, ...rest } = options.

Rest in Destructuring

Object Rest

const user = { id: 1, name: "Alice", age: 30, role: "admin" }

// Extract id, gather everything else
const { id, ...profile } = user
// id = 1
// profile = { name: "Alice", age: 30, role: "admin" }

// This is a clean way to REMOVE a property immutably:
const { role, ...userWithoutRole } = user
// userWithoutRole = { id: 1, name: "Alice", age: 30 }
// "role" is extracted but unused — effectively deleted
Tip: "Delete a property without mutating" pattern: const { propToRemove, ...rest } = obj. Use rest as the new object. ESLint may warn about the unused variable; prefix with _: const { role: _role, ...rest } = obj.

Array Rest

const [head, ...tail] = [1, 2, 3, 4]
// head = 1, tail = [2, 3, 4]

const [first, second, ...remaining] = "hello".split("")
// first = "h", second = "e", remaining = ["l", "l", "o"]

Immutable Update Patterns (React)

Why immutable updates?

React uses reference equality (===) to detect changes. If you mutate an object directly, its reference doesn't change, so React won't re-render. You must create a new object/array for React to detect the change.

Mutating (broken in React)
// WRONG — mutates in place
user.name = "Bob"
setUser(user) // same reference!
// React says "same object, skip re-render"

// WRONG — push mutates the array
items.push(newItem)
setItems(items) // same reference!
Immutable (correct in React)
// RIGHT — new object via spread
setUser({ ...user, name: "Bob" })
// New reference, React re-renders

// RIGHT — new array via spread
setItems([...items, newItem])
// New reference, React re-renders

Updating Nested State

type State = {
  user: {
    name: string
    address: {
      city: string
      zip: string
    }
  }
  settings: {
    theme: string
  }
}

// Update deeply nested city
const updateCity = (state: State, newCity: string): State => ({
  ...state,
  user: {
    ...state.user,
    address: {
      ...state.user.address,
      city: newCity
    }
  }
})

// This is verbose — it's why libraries like Immer exist
// Immer lets you write "mutating" code that produces
// immutable updates under the hood
Pain point: Deeply nested immutable updates are verbose. In real projects, you'll use Immer (built into Redux Toolkit) which lets you write state.user.address.city = newCity and produces the immutable update automatically.

Spread in Function Calls

const nums = [1, 5, 3, 9, 2]

// Spread array as individual arguments
Math.max(...nums)  // 9  — same as Math.max(1, 5, 3, 9, 2)

// Python equivalent: max(*nums)

// Spread into JSX props
const buttonProps = { disabled: true, className: "btn" }
// <button {...buttonProps}>Click</button>
// Same as: <button disabled={true} className="btn">Click</button>

Common Mistakes & Gotchas

Shallow copy only
const original = {
  name: "Alice",
  tags: ["admin", "user"]
}
const copy = { ...original }

copy.tags.push("editor")
// original.tags is ALSO modified!
// Spread only copies one level deep

// Fix: spread nested too
const deepCopy = {
  ...original,
  tags: [...original.tags]
}
Spread order matters
// defaults FIRST, overrides LAST
const good = { ...defaults, ...overrides }

// If reversed, overrides get clobbered
const bad = { ...overrides, ...defaults }
// defaults win! Not what you want.

// Same as Python:
// {**defaults, **overrides} = overrides win
Can't spread into an object from an array
// This works — index becomes key
const obj = { ...["a", "b"] }
// { 0: "a", 1: "b" } — probably not useful

// You probably wanted:
const arr = ["a", "b"]
const obj = { items: arr }
// or use Object.fromEntries()
Rest must be last
// Rest element must be the last one
const { a, ...rest, b } = obj  // ERROR!
const [first, ...rest, last] = arr // ERROR!

// Python allows *rest in the middle:
# first, *rest, last = [1,2,3,4]  ✓
// JS/TS does NOT allow this

Practical React Patterns

Forward props to child components
type CardProps = {
  title: string
  highlighted?: boolean
} & React.HTMLAttributes<HTMLDivElement>

const Card = ({ title, highlighted, ...divProps }: CardProps) => (
  <div
    {...divProps}
    className={`card ${highlighted ? "highlighted" : ""} ${divProps.className ?? ""}`}
  >
    <h3>{title}</h3>
    {divProps.children}
  </div>
)

// Usage: <Card title="Hello" highlighted onClick={handler} style={{...}} />
// title and highlighted are consumed; onClick, style forwarded to <div>
Managing list state with spread
const [todos, setTodos] = useState<Todo[]>([])

// Add
const addTodo = (text: string) =>
  setTodos(prev => [...prev, { id: crypto.randomUUID(), text, done: false }])

// Remove
const removeTodo = (id: string) =>
  setTodos(prev => prev.filter(t => t.id !== id))

// Toggle
const toggleTodo = (id: string) =>
  setTodos(prev => prev.map(t =>
    t.id === id ? { ...t, done: !t.done } : t
  ))
