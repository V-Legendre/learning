Figma Basics for Developers

What you need to know to read designs, extract values, and work with designers

What is Figma?

The short version

Figma is a browser-based collaborative design tool — think "Google Docs for UI design." No install needed, real-time multiplayer, everyone looks at the same live file.

It's the single source of truth for how the UI should look. You never get sent a static mockup — you always reference the live Figma file.

Backend analogy

Figma is to frontend what a shared database schema is to backend — the canonical spec that both designers and developers reference.

Designers write to it, developers read from it. Dev Mode makes it read-only for you so you can't accidentally break anything.

Frames vs Groups

Almost everything in Figma is a Frame. Understanding this is key.

Frame (the important one)
  • Has its own dimensions (W x H)
  • Can have fills, borders, corner radius
  • Supports Auto Layout (= flexbox)
  • Can clip overflow content
  • Children can have constraints

CSS analogy: a <div> with styles

Group (just convenience)
  • Bounds auto-adjust to fit children
  • Cannot have its own styling
  • No auto layout support
  • Cannot clip content
  • No constraints

CSS analogy: a transparent wrapper with no styles

Dev Mode (your main tool)

Toggle with Shift + D. This is where you'll spend your time in Figma.

What you see
  • Code tab — auto-generated CSS for the selected element
  • Spacing values — hover between elements to see distances
  • Typography — font family, size, weight, line-height
  • Colors — hex, RGB, HSL values
  • Component properties — variant, text, boolean props
  • Export section — download SVGs, PNGs directly
What you do
  • Click an element → read its CSS in the right panel
  • Hold Alt + hover → measure distance to other elements
  • Click on a component instance → see its name and props
  • Right-click → Copy as SVG for quick icon export
  • Canvas is read-only — you can't break anything
The #1 shortcut: Select any element, hold Alt/Option, then hover over other elements. You'll see the pixel distance between them. This is how you extract spacing values efficiently.

Design Tokens / Variables

Figma Variables = CSS custom properties. They're named values that the whole design references instead of raw hex codes.

Token hierarchy (same pattern as your code)
Primitive red-500 = #EF4444  |  space-4 = 16px — raw values
↓
Semantic color/error = {red-500}  |  spacing/md = {space-4} — purpose-based aliases
↓
Component button/bg/danger = {color/error} — component-specific tokens
Modes enable theming: a variable like color/bg/surface can be #FFFFFF in light mode and #1A1A2E in dark mode. Same variable name, different values per context.

  • Right-click any element → Copy as SVG (instant clipboard copy)
  • Right-click → Copy as PNG
  • Select element → right panel → Export section → choose format and scale → download
  • Cmd + Shift + E → batch export everything with export settings
